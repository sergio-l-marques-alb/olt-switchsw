SNMP Trap Configuration
Use the SNMP Trap Configuration page to configure whether or not SNMP traps are sent from the Captive Portal and to specify captive portal events that will generate a trap.
 
Note: You can configure the Captive Portal traps only if the Captive Portal Trap Mode is enabled, which you configure on the System > Trap Manager > Trap Flags > Trap Flags Configuration page.
All CP SNMP traps are disabled by default.
Field
Description
Captive Portal Trap Mode
Displays the current captive portal trap mode status.
Client Authentication Failure Traps
If you enable this field, the SNMP agent sends a trap when a client attempts to authenticate with a captive portal but is unsuccessful.
Client Connection Traps
If you enable this field, the SNMP agent sends a trap when a client authenticates with and connects to a captive portal.
Client Database Full Traps
If you enable this field, the SNMP agent sends a trap each time an entry cannot be added to the client database because it is full.
Client Disconnection Traps
If you enable this field, the SNMP agent sends a trap when a client disconnects from a captive portal.
