DiffServ Policy Help

Help for Diffserv Policy Configuration

Selection Criteria

    • Policy Selector - Along with an option to create a new policy, this lists all the existing DiffServ policy names, from which one can be selected. The content of this screen varies based on the selection of this field. If an existing policy is selected then the screen will display Member Classes for that DiffServ policy. If 'create' is selected, another screen appears to facilitate creation of a new policy. The default is 'create'.
    • Policy Type - Only when a new policy is created, this field is a selector field. After policy creation this becomes a non-configurable field displaying the configured policy type.
    • Available Class List - This lists all existing DiffServ class names, from which one can be selected. This field is a selector field only when a new policy class instance is to be created. After creation of the policy class instance this becomes a non-configurable field.
    • Member Class List - This lists all existing DiffServ classes currently defined as members of the specified Policy, from which one can be selected. This list is automatically updated as a new class is added to or removed from the policy. This field is a selector field only when an existing policy class instance is to be removed. After removal of the policy class instance this becomes a non-configurable field.

Configurable Data

    • Policy Name - This is a case-sensitive alphanumeric string from 1 to 31 characters uniquely identifying a policy.

Non-Configurable Data

    • Policy Type - Only when a new policy is created, this field is a selector field. After policy creation this becomes a non-configurable field displaying the configured policy type.
    • Member Class List - Displays all the member classes for the selected DiffServ policy. It is automatically updated as a new class is added to or removed from the policy. Only when an existing policy class instance is to be removed, this field is a selector field. After removal of the policy class instance this becomes a non-configurable field.
    • Available Class List - Displays all the member classes for the specified policy. It is automatically updated as a new class is added to or removed from the policy. Only when a new policy class instance is to be created this field is a selector field. After creation of the policy class instance this becomes a non-configurable field.

Command Buttons

    • Submit - Send the updated screen to the switch and cause the changes to take effect on the switch but these changes will not be retained across a power cycle unless a save is performed.
    • Delete - Delete the currently selected filter.
    • Rename - Allows to rename a specified policy.
    • Add Selected Class - Creates a policy class instance by attaching the policy to the specified class.
    • Remove Selected Class - Removes a policy class instance by detaching the policy from the specified class.

Help for Policing Attributes

Selection Criteria

    • Color Mode - This lists the color mode. The default is 'Color Blind'.
      • Color Blind
      • Color Aware

      Color Aware mode requires the existence of one or more color classes that are valid for use with this policy instance. A valid color class contains a single, non-excluded match criterion for one of the following fields (provided the field does not conflict with the classifier of the policy instance itself):

      • CoS
        •
      • IP DSCP
        •
      • IP Precedence
        •
      • Secondary CoS
        •
    • Conform Action Selector - This lists the actions to be taken on conforming packets per the policing metrics, from which one can be selected. The default is 'send'.
    • Exceed Action Selector - This lists the actions to be taken on excess packets per the policing metrics, from which one can be selected. The default is 'drop'.
      •
    • Violate Action Selector - This lists the actions to be taken on violating packets per the policing metrics, from which one can be selected. The default is 'drop'.
    • For each of the above Action Selectors one of the following actions can be taken:
      • Drop - These packets are immediately dropped.
      • Mark IP DSCP - These packets are marked by DiffServ with the specified DSCP value before being presented to the system forwarding element. This selection requires that the DSCP value field be set.
      • Mark CoS - These packets are marked by DiffServ with the specified CoS value before being presented to the system forwarding element. This selection requires that the Mark CoS value field be set.
      • Mark Secondary CoSP - These packets are marked by DiffServ with the specified CoS2 value before being presented to the system forwarding element. This selection requires that the Mark Secondary CoS value field be set.
        •
      • Send - These packets are presented unmodified by DiffServ to the system forwarding element.
      • Mark IP Precedence - These packets are marked by DiffServ with the specified IP Precedence value before being presented to the system forwarding element. This selection requires that the Mark IP Precedence value field be set.

Non-Configurable Data

    • Policy Name - Displays name of the specified DiffServ policy.
    • Policy Type - Displays type of the specified policy as .
    • Class Name - Displays name of the DiffServ class to which this policy is attached.
    • Policing Style - Displays the selected policing style: Simple, SingleRate, or TwoRate.

Command Buttons

    • Confirm - Provides the policing configuration screen.
    • Cancel - Cancel the currently selected filter.

Help for Policing Configuration

Selection Criteria

    • Color Conform Class - Visible only if the user has selected 'Color Aware' Color Mode on the Policing Attributes screen, this lists the DiffServ classes that are valid for use as a conform color-aware specifier. One of the classes must be selected from this list. A color-aware mode and value defined by the Color Conform Class must be different from the Color Exceed Class.
    • Color Exceed Class - Visible only if the user has selected 'Color Aware' Color Mode on the Policing Attributes screen for a Policing Style of 'Police Single Rate' or 'Police Two Rate', this lists the DiffServ classes that are valid for use as an exceed color-aware specifier. One of the classes may optionally be selected from this list. A color-aware mode and value defined by the Color Exceed Class must be different from the Color Conform Class. If no class is selected, the exceed color is not used.
    • Conform DSCP Keyword - This lists the keywords for the known DSCP values from which one can be selected. The list includes 'other' as an option for the remaining values.
    • Exceed DSCP Keyword - This lists the keywords for the known DSCP values from which one can be selected. The list includes 'other' as an option for the remaining values.
      •
    • Violate DSCP Keyword - This lists the keywords for the known DSCP values from which one can be selected. The list includes 'other' as an option for the remaining values.

Configurable Data

    • Conform IP Precedence Value - This is an IP Precedence value in the range 0 to 7.
    • Exceed IP Precedence Value - This is an IP Precedence value in the range 0 to 7.
      •
    • Violate IP Precedence Value - This is an IP Precedence value in the range 0 to 7.
    • Committed Rate - This value is specified in the range 1 to 4294967295 kilobits-per-second (Kbps). The committed rate is used to monitor arrival rate of incoming packets for this class.
    • Committed Burst Size - This value is specified in the range 1 to 128 KBytes. The committed burst size is used to determine the amount of conforming traffic allowed.
    • Excess Burst Size - This value is specified in the range 1 to 128 KBytes. The excess burst size is used to determine the amount of excess traffic allowed. This value must be equal to or greater than the committed burst size.
    • Peak Rate - This value is specified in the range 1 to 4294967295 kilobits-per-second (Kbps). The peak rate is used to monitor arrival rate of incoming packets for this class.
    • Peak Burst Size - This value is specified in the range 1 to 128 KBytes. The peak burst size is used in determining the amount of exceeding traffic allowed. This value must be equal to or greater than committed burst size.
      •

Help for Random Drop

Configurable Data

    • Minimum Threshold - This is an average queue depth minimum threshold in the range 1 to 250000 bytes.
    • Maximum Threshold - This is an average queue depth maximum threshold in the range 1 to 500000 bytes. This value must be equal to or greater than minimum threshold.
    • Maximum Drop Probability - This is the packet drop probability expressed as a percentage (0 to 100 %) when queue depth average reaches the maximum threshold.
    • Sampling Rate - This is the period in the range 0 to 1000000 microseconds, at which the queue depth is sampled for computing an average. A value of 0 means the sampling is performed for every packet. The default value is 0.
    • Decay Exponent - This is an integer value in the range 0 to 16 that determines how quickly the average queue depth calculation decays over time, with a higher number causing a slower decay rate. The default value is 9.

Non-Configurable Data

    • Policy Name - Displays name of the specified DiffServ policy.
    • Policy Type - Displays type of the specified policy as .
    • Class Name - Displays name of the DiffServ class to which this policy is attached.
