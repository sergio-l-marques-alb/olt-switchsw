Help for Loopback Test Result

Selection Criteria

    • Unit/Slot/Port Slot/Port - This field indicates the interface on which the Loopback Test is run.

Non-Configurable Data

    • Interface - Displays the interface tested in the Unit/Slot/Port Slot/Port notation. This field is displayed after the "Run" button has been clicked and results are available.
    • Loopback Status - This displays the Loopback status as Good or Bad based on Error condition on the port.
      Tested: the Port selected is Tested.
      Not-Tested: the Port selected is not-tested.
      Status of the Test: Good means the TX and RX direction of the Port is working Correctly on tested Layer. Bad means port is working incorrectly either in TX or RX based on Errors on the direction.

Command Buttons

    • Refresh - Refreshed test result page on the selected interface. Since the Loopback test may take up to 5 seconds to complete and also involves hardware intraction, user need to click "Refresh" Button to view the statistics like Data TX, RX and Error TX and RX etc.
    • Clear - Clears the Loopback test results page on the selected interface and go to original Loopback Test page.
