Help for MLD Routing Interface Configuration

Selection Criteria

    • Unit/Slot/Port Slot/Port - Select the interface for which data is to be displayed or configured from the pull down menu. You must have configured at least one router interface before configuring or displaying data for an MLD interface, otherwise an error message will be displayed.

Configurable Data

    • Interface Mode - Select enable or disable from the pull down menu to set the administrative status of MLD on the selected interface. The default value is Disable.
    • Version - Enter the version to be configured on the selected interface. Valid values are (V1 and V2). The default value is V2.
    • Query Interval (secs) - Enter the frequency in seconds at which MLD host-query packets are to be transmitted on this interface. Valid values are from . The default value is .
    • Query Max Response Time (milli-secs) - Enter the maximum query response time to be advertised in MLDv2 queries on this interface, in milli-seconds. Valid values are from . The default value is milli-seconds.
    • Last Member Query Interval (milli-secs) - Enter the last member query interval in milli-seconds. This is the maximum response time to be inserted into group-specific queries sent in response to leave group messages, and is also the amount of time between group-specific query messages. Valid values are from . The default value is milli-seconds.
    • Last Member Query Count - Enter the number of queries to be sent on receiving a leave group report. Valid values are from . The default value is .

Non Configurable Data

    • Robustness - The robustness parameter for the selected interface. This variable allows tuning for the expected packet loss on a subnet. If a subnet is expected to be losing, the robustness variable may be increased. MLD is robust to (robustness variable-1) packet losses. The default value is 2.
    • Startup Query Interval (secs) - This value indicates the configured interval (in seconds) between General Queries sent by a querier on startup.
      • between General Queries sent by a Querier on startup. The default value is 31 seconds.
    • Startup Query Count - This value indicates the configured number of Queries sent out on startup, separated by the Startup Query Interval. The default value is 2.

Command Buttons

    • Submit - Sends the updated configuration to the router. Configuration changes take effect immediately. These changes will not be retained across a power cycle unless a save is performed.
