Help For RIP Route Redistribution Configuration

Selection Criteria

    • Source - This select box is a dynamic selector used to configure Source Routes. The valid values are
      • Connected
      • Static
      • OSPF
        •
      • BGP
        •

Configurable Data

    • Redistribute - Select to Enable/Disable the route-redistribution for a particular source protocol.By default this is disable as none of the routes learned through other protocols including connected are redistributed.
    • Metric-Sets the metric value to be used as the metric of redistributed routes. This field is 0 if a metric has not been configured. The range of valid values is (1 to 15).
    • One or more of the following checkboxes must be selected to set the type of OSPF routes to be redistributed. These fields would appear only if Source is "OSPF". These fields display the configured match options if "OSPF" was pre-configured and can be modified.By default Match Internal Routes will be selected.
      • Match Internal Routes - Redistributes Internal Routes.
      • Match External Type 1 Routes - Redistributes External Type 1 Routes.
      • Match External Type 2 Routes - Redistributes Match External Type 2 Routes.
      • Match NSSA-External Type 1 Routes - Redistributes Match NSSA External Type 1 Routes.
      • Match NSSA-External Type 2 Routes - Redistributes Match NSSA External Type 2 Routes.
      •
    • Distribute List - Distribute List - Sets the Access List that filters the routes to be redistributed by the destination protocol. Only permitted routes are redistributed. If this command refers to a non-existent access list, all routes are permitted.This field is 0 if a distribute list has not been configured. The range of valid values is (1 to 199). When used for route filtering, the only fields in an access list that get used are
      • Source IP Address and netmask
      • Destination IP Address and netmask
      • Action (permit or deny)
      All other fields (source and destination port, precedence, tos, etc.) are ignored.
      The source IP address is compared to the destination IP address of the route. The source IP netmask in the access list rule is treated as a wildcard mask, indicating which bits in the source IP address must match the destination address of the route. (Note that a 1 in the mask indicates a "don’t care" in the corresponding address bit.)
      When an access list rule includes a destination IP address and netmask (an extended access list), the destination IP address is compared to the network mask of the destination of the route. The destination netmask in the access list serves as a wildcard mask, indicating which bits in the route’s destination mask are significant for the filtering operation.
