1. port

1.1. port > general

    Overview

    General attributes

    Property Description Synopsis Default Device
    load_firmware
    The method of the firmware load.
    2 - fast; 1 - MDIO load; 0 - do not load firmware

    		 BCM88790_A0
    load_firmware
    Firmware load verification method
    The eight bit of the property value enables firmware CRC calculation

    		 BCM88790_A0
    fabric_mac_bucket_fill_rate
    Mac bucket fill rate.
    Bucket-Fill-Rate. Number of good cells that add a token to the bucket.
    BktFillRate 3:0 must be configured to be no bigger than 0xB.
    The number BktFillRate 3:0 is the exponent of 2,
    i.e., the number of good cells adding a token to the bucket is 2^BktFillRate 3:0.
    Good cell is defined according to RxCntCfg

    		 fabric_mac_bucket_fill_rate=<[0 - 11]> 6 BCM88790_A0

1.3. port > static_add

    Overview

    General static add information

    Property Description Synopsis Default Device
    port_init_speed
    Port data speed.
    To set different port rate use:
    port_init_speed_#port#=#speed#
    Speed value -1 is ALLOWED.
    If speed is set to be -1 this will boot the device with the port disabled.

    		 port_init_speed=<-1 | 10312 | 11500 | 20625 | 23000 | 25000 | 25781 | 50000 | 53125 | 56250> BCM88790_A0
    port_init_cl72
    Enable / Disable CL72
    port_init_cl72_#port# = 0 / 1

    		 port_init_cl72=<0 | 1> BCM_PORT_RESOURCE_DEFAULT_REQUEST BCM88790_A0
    port_fec
    Specifies the FEC type. The types are as follows:
    0 - no FEC
    1 - BASE R - 64/66b KR FEC
    5 - RS 206 - 64/66b 5T RS FEC
    6 - RS 108 - 64/66b 5T low latency RS FEC
    7 - RS 545 - 64/66b 15T RS FEC
    8 - RS 304 - 64/66b 15T low latency RS FEC

    		 port_fec=<0 | 1 | 5 | 6 | 7 | 8> BCM88790_A0
    phy_tx_polarity_flip
    Flips PHY TX polarity if enabled
    phy_tx_polarity_flip_#port# = 0 / 1

    		 phy_tx_polarity_flip=<0 | 1> 0 BCM88790_A0
    phy_rx_polarity_flip
    Flips PHY RX polarity if enabled
    phy_rx_polarity_flip_#port# = 0 / 1

    		 phy_rx_polarity_flip=<0 | 1> 0 BCM88790_A0
    repeater_link_enable
    Shows if a certain FE link is connected to a device in reteamer mode

    		 repeater_link_enable=<0 | 1> 0 BCM88790_A0
    port_tx_pam4_precoder
    is the precoding enabled on TX side.
    port_tx_pam4_precoder_#port#=enable/disable

    		 port_tx_pam4_precoder=<enable | disable> disable BCM88790_A0
    port_lp_tx_precoder
    has the link partner enabled pre-coding on its TX side.
    in other words - enable the decoding on my RX side.
    port_lp_tx_precoder_#port#=enable/disable

    		 port_lp_tx_precoder=<enable | disable> disable BCM88790_A0
    serdes_lane_config_dfe
    DFE filter
    serdes_lane_config_dfe_#port#=on|off|lp
    on - dfe is on and lp dfe is off
    off - both dfe and lp dfe are off
    lp - both dfe and lp dfe are on

    		 serdes_lane_config[_dfe<port>]=<on | off | lp> BCM88790_A0
    serdes_lane_config_media_type
    media type
    serdes_lane_config_media_type_#port#=backplane|copper|optics

    		 serdes_lane_config[_media_type<port>]=<backplane | copper | optics> BCM88790_A0
    serdes_lane_config_unreliable_los
    unreliable los
    serdes_lane_config_unreliable_los_#port#=0|1

    		 serdes_lane_config[_unreliable_los<port>]=<0 | 1> DNXC_PORT_SERDES_LANE_CONFIG_INVALID_VAL BCM88790_A0
    serdes_lane_config_cl72_auto_polarity_en
    enable cl72 auto polarity
    serdes_lane_config_cl72_auto_polarity_en_#port#=0|1

    		 serdes_lane_config[_cl72_auto_polarity_en<port>]=<0 | 1> DNXC_PORT_SERDES_LANE_CONFIG_INVALID_VAL BCM88790_A0
    serdes_lane_config_cl72_restart_timeout_en
    enable cl72 restart timeout
    serdes_lane_config_cl72_restart_timeout_en_#port#=0|1

    		 serdes_lane_config[_cl72_restart_timeout_en<port>]=<0 | 1> DNXC_PORT_SERDES_LANE_CONFIG_INVALID_VAL BCM88790_A0
    serdes_lane_config_channel_mode
    channel mode
    serdes_lane_config_channel_mode_#port#=force_nr|force_er

    		 serdes_lane_config[_channel_mode<port>]=<force_nr | force_er> BCM88790_A0
    serdes_tx_taps
    a set of TX FIR parameters for the port.
    serdes_tx_taps_#port#=signalling_mode:pre:main:post:pre2:post2:post3
    signalling_mode can be either pam4 or nrz.
    [pre2:post2:post3] part is optional:
    - if it is given - this it 6-tap mode.
    - if it is not given - this is 3-tap mode.
    main is set to DNXC_PORT_TX_FIR_INVALID_MAIN_TAP by default to distinguish between whether the SoC property was set or not.

    		 BCM88790_A0
    serdes_qrtt_active
    Enable / Disable Quad

    		 serdes_qrtt_active=<0 | 1> 1 BCM88790_A0

3.3. fabric > multicast

    Overview

    General multicast attributes

    Property Description Synopsis Default Device
    fe_mc_id_range
    Specifies the MC IDs range.
    1. 64K - Supports eight queries at each clock, maximum of 64K multicast IDs, and up to 192 next-stage devices.
    2. 128K_HALF - Supports eight queries at each clock, maximum of 128K multicast IDs, and up to 96 next- stage devices.
    3. 128K - Supports four queries at each clock, maximum of 128K multicast IDs, and up to 192 next-stage devices.
    4. 256K_HALF - Supports four queries at each clock, maximum of 256K multicast IDs, and up to 96 next- stage devices.
    5. 256K - Supports two queries at each clock, maximum of 256K multicast IDs, and up to 192 next-stage devices.
    6. 512K_HALF - Supports two queries at each clock, maximum of 512K multicast IDs, and up to 96 next- stage devices.

    		 fe_mc_id_range=<64K | 128K_HALF | 128K | 256K_HALF | 256K | 512K_HALF> 256K_HALF BCM88790_A0
    fabric_multicast_mode
    Specifies the MC mode.
    Direct multicast is valid in a single-stage systems with a highest FAP-ID less then 192/96 (according to MC table mode).

    		 fabric_multicast_mode=<DIRECT | INDIRECT> DIRECT BCM88790_A0
    fe_mc_priority_map_enable
    Enable / Disable mapping internal multicast priority according to MC-ID ranges.
    1: Map MC-ID to fabric priority.
    0: Extract MC cell priority from cell header.

    		 fe_mc_priority_map_enable=<0 | 1> 0 BCM88790_A0

4. linkscan

4.1. linkscan > general

    Overview

    general data

    Property Description Synopsis Default Device
    bcm_linkscan_interval
    Configure linkscan interval in microseconds
    bcm_linkscan_interval= 0~0xffffffff
    Default: 250,000

    		 bcm_linkscan_interval=<[0 - 4294967295]> 250000 BCM88790_A0
    bcm_linkscan_errdelay
    The amount of time in microseconds for which the bcm_linkscan module will suspend
    a port from further update processing after 'bcm_linkscan_maxerr' errors are detected.
    After this delay, the error state for the port is cleared and normal linkscan processing
    resumes on the port
    bcm_linkscan_errdelay= 0~0xffffffff
    Default: 10,000,000

    		 bcm_linkscan_errdelay=<[0 - 4294967295]> 10000000 BCM88790_A0
    bcm_linkscan_maxerr
    The number of port update errors which will cause the bcm_linkscan module to suspend
    a port from update processing for the period of time set in 'bcm_linkscan_errdelay'
    bcm_linkscan_maxerr= 0~0xffffffff
    Default: 5

    		 bcm_linkscan_maxerr=<[0 - 4294967295]> 5 BCM88790_A0
    linkscan_thread_pri
    Specifies the priority of the BCM Linkscan thread
    linkcan_thread_pri= 0~0xffffffff
    Default: 50

    		 linkscan_thread_pri=<[1 - 99]> 50 BCM88790_A0
