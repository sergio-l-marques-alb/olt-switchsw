1. General

Overview

This document presents description of the CTests. All commands are organized in ctest tree, where leaves are actual executable tests. Manual presents description, synopsis, arguments and examples per each leaf. Each argument has certain type which defines input format, as may be see from argument description. Argument may have default and thus not required on command line, if there is no default(tagged by 'None' in usage). user must provide it. All arguments should be provided in arg=<value> paradigm with 2 exceptions: 1. Boolean argument provided without value imply 'TRUE', 2. Argument with 'free' tag allows omitting argument name, providing values only. Values without argument names will be associated with free argument according to the order. 1st nameless argument to the 1st free one, 2nd to the second and so on Besides the arguments presented per command, there are system arguments that may be used with any leaf, There are also system commands which may be invoked after each branch or leaf, see details below

1.1. General > list

Description

List available ctests

XML Output Support

No

Synopsis

LIST ctest [commands] list [pre] [rnd] [label] [system_mode]

Arguments

# Name Type Default Description
1. pre boolean No Include only tests with precommit flag
2. random boolean No Include only tests with random flag
3. label string EMPTY Include tests with specific label
4. system_mode string EMPTY Include tests with specific system mode

Examples

LIST ct list

LIST ct tm list pre

LIST ct export label=kbp

LIST ct export system_mode=kbp

1.2. General > run

Description

Execute ctests

XML Output Support

No

Synopsis

RUN ctest [commands] run [pre] [rnd] [lg] [fail] [um]

Arguments

# Name Type Default Description
1. logger boolean No Set logger mode
2. failed boolean No Results table will show only failed tests
3. pre boolean No Include only tests with precommit flag
4. random boolean No Include only tests with random flag

Examples

RUN ct run

RUN ct tm run pre

Description

Insert, update and delete L3 Host IPv4 entries while recording how much time each step takes and outputs and/or validates the entry rates

Synopsis

Arguments

# Name Type Default Description
1. Entries uint32 10000 Number of entries
2. Runs uint32 10 Number of runs
3. Update boolean YES Indication to perform update measurements
4. DeLeTe boolean YES Indication to perform delete measurements
5. Random boolean YES Indication to validate measurements with random entries
6. Validate boolean NO Indication to validate the measurements
7. Update_measures boolean NO Indication to update the measurements
8. Public boolean NO Indication to validate the measurements of public routes
9. RPF boolean NO Indication to validate the measurements on Route RPF table
10. TCAM boolean NO Indication to validate the measurements IPMC TCAM table
11. EM boolean NO Indication to validate the measurements IPMC EM table
12. Layer string BCM Select starting layer

Examples

3. mdb

Overview

Test the MDB physical table operation.

Arguments

# Name Type Default Description
1. Table string EMPTY DBAL physical table name
2. MoDe int32 0 The test mode: 0-fast (tests ~1/10 of the capacity), 1-full(direct)/capacity test (EM/LPM) 2-capacity test mixed entry encoding (EM) 3-MDB iterators test(SW and HW) 4-SER KAPS TCAM only
3. all boolean No Ignore Table and run the test on all table
4. DeLeTe boolean Yes Perform a delete operation as part of the test, relevant for EM and LPM
5. DYNAMIC int32 0 Run a dynamic variation of the test. For EEDB: 1-dynamic EEDB bank, 2-dynamic MDB cluster
6. app_id uint32 0xFFFFFFFF The APP ID
7. FULL boolean No Include entry preparation time and descriptor DMA (if enabled) commit to HW in the time measurement.

Examples

CTest MDB all

CTest MDB table=LEM mode=1
