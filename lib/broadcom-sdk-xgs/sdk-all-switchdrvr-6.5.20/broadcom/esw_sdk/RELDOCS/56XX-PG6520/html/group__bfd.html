◆ bcm_bfd_endpoint_create()

int bcm_bfd_endpoint_create ( int  unit,
bcm_bfd_endpoint_info_t *  endpoint_info 
)

Create or update an BFD endpoint object.

If BCM_BFD_ENDPOINT_UPDATE is not set, a new BFD endpoint is created. If BCM_BFD_ENDPOINT_WITH_ID is not set, an ID is created for the endpoint. Otherwise, the specified ID is used. If BCM_BFD_ENDPOINT_UPDATE is set, the endpoint with the specified ID is updated as specified in the structure.

Parameters
[in]unitUnit number.
[in,out]endpoint_infoPointer to an BFD endpoint structure.

◆ bcm_bfd_status_multi_get()

int bcm_bfd_status_multi_get ( int  unit,
int  max_endpoints,
bcm_bfd_status_t *  status_arr,
int *  count 
)

User should call this API - once to retrieve endpoint count and again to retrieve the actual endpoint status.

If user passes max_endpoints as 0, the API returns the number of endpoints currently configured in the system via the "count" parameter.User can use this value to declare status_arr of "count" time its size and pass the same to API. User should pass count (obtained as mentioned in previous point) as max endpoints to the API to get the session status.

Parameters
[in]unitUnit number.
[in]max_endpointsmax no of endpoints.
[out]status_arrPointer to all Session required data
[out]countPointer to number of actual endpoints created
