Port Abilities

Each port known to the API has certain abilities. These abilities are determined during the initialization of the port module. For a given port, The overall capabilities for a port are determined by taking the least common denominator of all the factors. These attributes can be broken by their functionality as follows:

  • PHY attributes, as determined by the PHY driver (fixed by underlying device for ports with integrated PHYs)
  • MAC attributes, as determined by the MAC driver (fixed by underlying device)
  • Attributes as specified by driver run-time configuration properties (see "Configuration Properties" in BCM56XXX Network Switching Platform Programmer's Guide).

Port Information Control Structure (bcm_port_info_t)

API calls are supplied to set and get port characteristics independently. However, it is often useful to set or get a group of characteristics for a port at the same time. Several functions in the port module accept a pointer to a port information structure or BCM_PORT_INFO_t . This structure is used as in input parameter on some calls and an output parameter on other calls.

Calls using this structure can be generally thought of as making a series of individual calls to the API for port or rate configuration. To allow a subset of the complete list of possible calls, the first field in BCM_PORT_INFO_t is an action mask (action_mask). This mask can be used to control which subset of the possible calls are performed. This applies to both setting and retrieving state. See bcm_port_info_get for information.

For both input and output usages, the first fields in the structure are action masks (action_mask, action_mask2). These masks indicate which operations should be performed (as well as which fields are valid). BCM_PORT_INFO_t describes the fields in the structure along with an indication of the mask value associated with that field.

FlexPort Support

FlexPort feature is supported on a number of devices such as BCM56639, BCM56840, BCM56850 and BCM56860. This feature allows some port blocks to operate either as a single port, or as two ports, or as four ports, etc. The hardware provides a mechanism to dynamically switching among these modes of operation without affecting other port blocks.

There are two set of APIs that can be used to perform a FlexPort operation:

  • Port Control bcmPortControlLanes: bcm_port_control_get and bcm_port_control_set
  • Port Resource APIs: bcm_port_resource_t_init, bcm_port_resource_get, bcm_port_resource_set, bcm_port_resource_multi_set, bcm_port_resource_traverse, these are currently supported in BCM56860.

It is not recommended to use both set of APIs. It is best to choose one set.

Port Control - bcmPortControlLanes

For each port block that supports this feature, a new control, bcmPortControlLanes, specifies the number of lanes available to the first port of that block, referred to as the controlling or master port. The remaining lanes will be assigned to other port(s) which use the same number of lane(s) as the master port. The following sequence must be followed to use this feature:

  • Use the bcm_port_subsidiary_ports_get API to get the list of ports that are members of the same port block as the given controlling port.
  • Use bcm_port_control_get for bcmPortControlLanes for the controlling port. If the value=1, all ports in the block are in use. If the value=4, this means only the controlling port is in use for the port block with four lanes
  • Disable all ports obtained in the previous step using bcm_port_enable_set with value = FALSE.
  • Remove the above ports from the counter DMA and linkscan bitmaps.
  • Call bcm_port_control_set with type bcmPortControlLanes for the controlling port with the desired value.
  • If the value=1, all ports in the block are in use. If the value=2, two ports are in use in a 4-lane port block. If the value=4, only the controlling port is in use in a 4-lane port block. Enable all ports that are in use by calling bcm_port_enable_set with value = TRUE.
  • Enable counter DMA and linkscan, if desired, for the newly enabled ports.

BCM56639 can operate either as a single high-speed port (up to 12G) or as 4 1G ports. BCM56840/BCM56850/BCM56860 can operate either as a single 40G port, or as two 20G ports, or as four 10G ports.

BCM5626x/BCM5646x front panel ports operate as a single 10G port or as four 2.5G ports. The back panel ports operate as one 10G port or two 10G ports or four 10G ports.

FlexPort Port Resource

The new BCM data structure bcm_port_resource_t and corresponding new set of FlexPort APIs bcm_port_resource_t_init, bcm_port_resource_get, bcm_port_resource_set, bcm_port_resource_multi_set, bcm_port_resource_traverse, allow logical to physical port number reassignment, as well as changes to the ports speed, number of PHY lanes, and encapsulation modes.

The ability to change port mappings allows applications using devices with fewer numbers of logical ports than physical ports, such as BCM56860, to access any physical port during run time.

In addition, applications can perform FlexPort operations in several port blocks with a single API call, rather than making several API calls to achieve a desired port configuration.

It is not recommended to mix these APIs with the legacy bcm_port_control_set (bcmPortControlLanes).

Applications should follow this sequence, similar to the one for the Port Control bcmPortControlLanes, to use this feature properly:

  • If needed, use the bcm_port_subsidiary_ports_get API to get the list of ports that are members of the same port block as the given controlling port.
  • Disable all ports that are involved in the FlexPort operation using bcm_port_enable_set with value = FALSE.
  • Remove all ports that are involved in the FlexPort operation from the counter DMA and linkscan bitmaps or any other application that refers to those ports.
  • Remove all references to any reassigned logical port number before port reconfiguration. If this is not performed, then the per-port properties and forwarding logic will retain the configuration of the previous personality of the logical port number and the behavior is undefined.
  • Perform the FlexPort operation by calling bcm_port_resource_multi_set, make sure the Port Resource array is populated properly as described in the API section.
  • Reconfigure switch logic if required. A new or remapped logical port created after a FlexPort operation will look like a port that is brought up after the SDK initialization.
  • Enable counter DMA and linkscan or any other application specific component, if desired, for the flexed ports.
  • Enable all flexed ports, if desired, by calling bcm_port_enable_set with value = TRUE.
