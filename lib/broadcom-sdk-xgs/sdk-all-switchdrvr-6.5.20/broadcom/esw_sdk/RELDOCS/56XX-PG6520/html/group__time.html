◆ bcm_time_bs_time_get()

int bcm_time_bs_time_get ( int  unit,
bcm_time_spec_t *  bs_time 
)

Retrieves broadsync time from device.

Retrieves broadsync time in seconds and nanoseconds format from the device. If the interface was configured in INPUT mode i.e. BroadSync slave the API will return the received time from the broadsync timecode signal. If the interface was configured as BroadSync master the API will return the output time for the current heartbeat.

Parameters
[in]unitUnit number.
[in,out]bs_timeA bcm_time_spec_t structure to fill

Register callback function for a given type.

Register callback function for a given type typedef int (*bcm_time_capture_cb)(int unit, void *user_data, uint32 *cb_data, bcm_time_capture_cb_type_t cb_status); The registered callbacks will be called whenever there are event timestamps available and/or when an overflow occurs in the FIFO. The cb_data buffer contains either the timestamps or overflow information as described below:

Callback data format
cb_status cb_data
bcmTimeCaptureOverflow Octet 0...1 : Event handle that caused overflow
bcmTimeCaptureFill Octet 0...1 : Number of timestamps captured Octet 2...3 : Event handle for timestamp1 Octet 4...10 : timestamp1 (52 bits) Octet 11...12 : Event handle for timestamp2 Octet 13...19 : timestamp2 (52 bits) ...
Parameters

Retrieves time capture from the device.

Retrieves time capture from the device, 3 different local clocks will be captured Free running clock Synchronous clock Syntonous clock If the interface was configured in INPUT mode than also clock sent by the external output device will be captured. Received clock Received accuracy Received clock is the clock sent with the heartbeat from external output device. BCM_TIME_CAPTURE_LOCKED flag indicates if the clock is locked on grand master's clock. Received accuracy indicates how accurate the clock provided by the output device.
