int bcm_rate_bandwidth_get ( int  unit,
bcm_port_t  port,
int  flags,
uint32 *  kbits_sec,
uint32 *  kbits_burst 
)

Configure/retrieve metering rate limit for specified packet type on given port.

Configure or retrieve the rate limiting status of a port for a specific packet type rounded to 64 kilobits multiplier. The flags argument indicates whether rate limiting is enabled for the desired packet type. The port argument is only relevant for devices which allow per-port rate limiting, such as StrataXGS switches. The kbits_sec as BCM_RATE_DISABLE(zero), disables the metering and specifying kbits_sec as BCM_RATE_BLOCK (-1), blocks all packets of that type. For selecting pps mode storm control rate limiting, BCM_RATE_MODE_PACKETS need to be specified along with the packet-type flags. Note: Specifying more then 1 packet-type flag in get function may lead to bogus results. Get API needs to be called with the same mode (Packets vs Bytes) as the set for fetching the configured rate limits.

Parameters
[in]unitUnit number.
[in]portDevice or logical port number
[in]flags(for "_set") Bitmask with one or more of BCM_RATE_* RateLimitFlags
(for "_get") Bitmask with one or more of BCM_RATE_* RateLimitFlags
[out]kbits_sec(for "_set") Rate limit value in kilobits (1000 bits)/second, value of 0 disables metering
(for "_get", OUT) Rate limit value in kilobits (1000 bits)/second, if 0 metering is disabled
[out]kbits_burst(for "_set") Rate limit value in kilobits (1000 bits)/second, value of 0 disables metering
(for "_get", OUT) Rate limit value in kilobits (1000 bits)/second, if 0 metering is disabled
Return values
BCM_E_NONESuccess.
BCM_E_UNAVAILNot supported.
BCM_E_XXXOther possible errors; for details, see error_codes

◆ bcm_rate_get()

int bcm_rate_get ( int  unit,
int *  pps,
int *  flags 
)

Configure/retrieve rate limit and on/off state of DLF, MCAST, and BCAST limiting.

bcm_rate_set configures global DLF, MCAST, and BCAST rate limiting for a BCM device. A flag in the flags argument selects whether rate limiting is to be enabled for the given packet type(s). The rate is set to the provided packets/second value for all ports on the BCM device. This routine assumes that the DLF, MCAST, and BCAST rates are to be set to the same value. bcm_rate_get retrieves the packets/second setting for broadcast packets. The DLF, MCAST, and BCAST rates must be are the same (if enabled). The flags argument returns the enabled status of rate limiting for the packet types. This function should be used in conjunction with bcm_rate_set, which sets the rate and enables state uniformly across the device.

Parameters
[in]unitBCM device number
[out]pps(for "_set") Rate limit value in packets/second
(for "_get", OUT) Rate limit value in packets/second
[out]flags(for "_set") Bitmask with one or more of BCM_RATE_* RateLimitFlags
(for "_get", OUT) RateLimitFlags
Return values
BCM_E_NONESuccess.
BCM_E_UNAVAILNot supported.
