Histogram data process is controlled via FlexCtr APIs. When the mode is selected, histogram data configuration and counter collecting APIs will not be supported.

◆ bcm_latency_monitor_register()

int bcm_latency_monitor_register ( int  unit,
int  monitor_id,
bcm_latency_monitor_callback_fn  fn,
void *  user_data 
)

The API is used by the application to register a callback routine that gets called by SDK when there is update of software ring buffer with histogram data. User application implements the callback handler and the processing in the callback handler should be minimal as the calling thread would be blocked when this callback is invoked.

The API is used by the application to register a callback routine that gets called by SDK when there is update of software ring buffer with histogram data. User application implements the callback handler and the processing in the callback handler should be minimal as the calling thread would be blocked when this callback is invoked.

struct bcm_latency_monitor_cb_info_s
{
int monitor_id; //Latency Monitor ID.
uint8 monitor_status; //Latency Monitor status.
void *sw_buf_addr; //Base address of 64 bits wide software ring buffer address.
uint32 sw_buf_size; //Size of 64 bits wide software ring buffer address.
uint32 sw_buf_wr_idx; //Index in to the software ring buffer pointing to last write operation.
} bcm_latency_monitor_cb_info_t;
