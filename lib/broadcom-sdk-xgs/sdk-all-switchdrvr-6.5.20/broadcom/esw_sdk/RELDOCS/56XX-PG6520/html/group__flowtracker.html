BCM45370 supports two checks as part of one hardware resource. The user can choose to do check on two separate attributes on single flow. e.g ttl>100 and ip_proto<10. Range operations require two separate checks and therefore if first check is created with range operation then second check can not be created. Other operations can be used to create two separate checks. please check bcm_flowtracker_check_operation_t for complete list of operations. The API takes bcm_flowtracker_check_info_t as input which has bcm_flowtracker_tracking_param_info_t to specify tracking parameter on which check will be created. BCM56370 support only the packet attributes in the range of 32bits as a parameter to check. e.g. bcmFlowtrackerTrackingParamTypePktCount and bcmFlowtrackerTrackingParamTypeByteCount can not be used as they specify the counts and not objects. similarly bcmFlowtrackerTrackingParamTypeInnerSrcIPv6 can not be used as it does not fall in 32bit range. bcm_flowtracker_check_operation_t contains the list of all operations which can be used to perform checks.
