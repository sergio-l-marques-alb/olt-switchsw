int bcm_qos_map_add ( int  unit,

Add a specific mapping entry to a QoS map.

A QoS map identified by map_id is a set of mapping entries between a packet's QoS markings and the internal priority and color. The bcm_qos_map_add API adds a specific mapping entry to the QoS map. This API actually programs the mapping in the hardware. Configuration flags which may be specified include L2 (untagged, outer, inner, vlan_pcp), L3 (ipv4, ipv6) or MPLS. The BCM_QOS_MAP_SUBPORT flag is used for mapping egress SubTag CoE packet's PHB (internal priority and color) to subport tag's Priority Code Point(PCP) and Drop Eligible(DE). If the paremeter flags = BCM_QOS_MAP_SUBPORT then map the egress SubTag CoE packet's internal priority and color to the subport tag's PCP and DE.

◆ bcm_qos_map_create()

int bcm_qos_map_create ( int  unit,
uint32  flags,
int *  map_id 
)

Allocates hardware resources for QoS mapping.

Allocates hardware resources for QoS mapping. Depending on the type of mapping specified by the configuration flags (L2, L3, L2_VLAN_PCP, MPLS, ingress, egress), the corresponding set of entries is reserved in the appropriate priority mapping table. The map_id is an identifier associated with the reserved chunk. A QoS map can be shared by multiple objects, which are GPORTS. The flags which are passed in indicate which type of map is to be created. These flags are or-ed together. General flags: BCM_QOS_MAP_WITH_ID - Use the indicated map table and type indicated in the map passed in. BCM_QOS_MAP_REPLACE - Replace the indicated map type. BCM_QOS_MAP_INGRESS - Create an ingress map used in conjunction with map type. BCM_QOS_MAP_EGRESS - Create an egress map used in conjunction with map type. L2 flags: BCM_QOS_MAP_L2 - create L2 type of map - defaults to outer tag map. For Trident2plus, this flag will only create an MPLS type of map while outer tag map will be created via flag BCM_QOS_MAP_L3. BCM_QOS_MAP_L2_OUTER_TAG - create L2 outer tag map. BCM_QOS_MAP_L2_INNER_TAG - create L2 inner tag map. BCM_QOS_MAP_L2_UNTAGGED - create L2 untagged map. BCM_QOS_MAP_L2_VLAN_PCP - indicates that mapping is done via VLAN_PCP rather than LIF. If not set and other L2 flags set, indicates LIF COS map type. MiM flags: BCM_QOS_MAP_MIM_ITAG - create a MiM map - used in conjunction with L2 flags. L3 flags: BCM_QOS_MAP_L3 - create L3 type of map - defaults to IPV4 map. For Trident2plus, this flag will create both IPV4 map and outer tag map. BCM_QOS_MAP_IPV4 - create IPV4 L3 map. BCM_QOS_MAP_IPV6 - create IPV6 L3 map. For the BCM56150, BCM56160 and BCM53400 families, these flags are not supported. To configure the ingress DSCP mapping, user can use port API bcm_port_dscp_map_set/get() to control related DSCP mapping. FCOE flags: BCM_QOS_MAP_VFT - create a FCOE VFT map. BCM_QOS_MAP_VSAN - create a FCOE VSAN map. BCM_QOS_MAP_RCH - create a FCOE VSAN map. MPLS BCM_QOS_MAP_MPLS - create an MPLS type of map. BCM_QOS_MAP_MPLS_ELSP - create an MPLS map for ELSP. BCM_QOS_MAP_MPLS_PHP - create an MPLS map for PHP. MiML flags: BCM_QOS_MAP_MIML - create a MiML map - used in conjunction with L2 flags. Other: BCM_QOS_MAP_ENCAP - indicates an encapsulated header. BCM_QOS_MAP_PACKET_INVALID Drops packets on a specific port*VLAN when the UP/PCP values do not match configured values for that port BCM_QOS_MAP_IGNORE_OFFSET This option prevents AC offset from being added to the default logical interface, although the offset still applies to Qos-related processing BCM_QOS_MAP_OPCODE Sets the AC according to the CoS attributes arriving in the packet. This map ID sets the opcode according to either the IPV4 or IPV6 TOS field or the Layer2 header according to the VLAN tag or resolving the packet using a combination of Traffic Class and Drop Precedence fields. BCM_OAM_MAP_OAM_PCP - Configures the mapping of the TOS to OAM-PCP. Must be used together with one (and only one) of the flags BCM_QOS_INGRESS or BCM_QOS_EGRESS BCM_QOS_MAP_OAM_INTPRI - Configures the mapping of the Internal priority to LM counter offset. BCM_QOS_MAP_OAM_OUTER_VLAN_PCP - Configures the mapping of the outer vlan priority to LM counter offset. BCM_QOS_MAP_OAM_INNER_VLAN_PCP - Configures the mapping of the inner vlan priority to LM counter offset. BCM_QOS_MAP_OAM_MPLS_EXP - Configures the mapping of the exp field in mpls label to LM counter offset. BCM_QOS_MAP_REPLICATION - Configures replication QOS map profile. This API need to be used in conjunction with the bcm_qos_map_add to build the desired mapping and then use the map_id. This is supported on 5662x/5663x/5664x/5645x/5685x/5686x/5687x/5637x/5677x/5697x family of devices. Note: Mixing the bcm_qos_map_create APIs with legacy individual APIs such as bcm_port_dscp_map_set, bcm_port_vlan_priority_map_set to create and bind different mappings such as L2/L3/VAN/PORT will result in undesirable behavior.
