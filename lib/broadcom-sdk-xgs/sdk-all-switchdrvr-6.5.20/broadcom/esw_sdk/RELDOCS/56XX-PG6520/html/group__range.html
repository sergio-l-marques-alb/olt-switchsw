◆ bcm_range_create()

int bcm_range_create ( int  unit,
int  flags,
bcm_range_config_t *  range_cfg 
)

Create a Range checker.

This API is to create a Layer 4 TCP/UDP port range checker or a VLAN ranger checker or a packet length range checker or a UDF range checker and returns an opaque bcm_range_t identifier that can be passed to 'bcm_field_qualify_RangeCheck' API, to qualify a Field entry with Range Checker match criteria. The required hardware resources are allocated and programmed. The range can be inverted using the 'invert' argument to bcm_field_qualify_RangeCheck. This API can optionally take "flags" from the user to decide whether to create a Range Checker with a specific bcm_range_t handle. After initializing the range_config using bcm_range_config_t_init API, User need to update required members (as mentioned below) of it before passing it to bcm_range_config_create API.

  • rid : If BCM_RANGE_CREATE_WITH_ID flag is passed to API, a valid rid should be given.
  • rtype : User need to specify type of range checker of type bcm_range_type_t to create. This is mandatory of all range checker types.
  • min : Lower range to match (inclusive) and mandatory for all range checker types.
  • max : Upper range to match (inclusive) and mandatory for all range checker types.
  • offset : This is applicable only for UDF range checkers (bcmRangeTypeUdf). This is the offset (in bits) for mask to apply before range check comparison with min and max values.
  • width : This is applicable only for UDF range checkers (bcmRangeTypeUdf). This is the width (in bits) from offset specified for mask to apply before range check comparison with min and max values. For example for a particular UDF range checker, offset is specified as 10, and width is specified as 6, then that means user intends to qualify on MSB 7 bits of 16 bit UDF chunk. These 7 bits gives values from 0 to 128 which will be range checked using min and max values specified.
  • udf_id : This is applicable only for UDF range checkers (bcmRangeTypeUdf). User needs to pass the udf id created using bcm_udf_create API. This value is required to know whether the UDF chunk used by this UDF can be used for range checking.
  • ports : This is a PBMP to specify for which pipe of XGS pipeline user wants to create this range checker. User can specify a specific pipe or all pipes. User needs to call bcm_port_config_get API to get 'per pipe'/all pipe PBMP. This parameter is mandatory for all range checker types.

◆ bcm_range_group_create()

int bcm_range_group_create ( int  unit,
bcm_range_group_config_t *  range_group_config 
)

Create a Range checker Group.

This API will help to combine different Range Checkers into a single group and returns a Range Check Group ID which can be qualified in a single IFP rule. This will help in reducing the number of rules for various range check conditions. Input to the API is a bitmap of RangeCheck IDs that need to be grouped together and the related ports for the pipe where group needs to be created. The Individual RangeCheck conditions and its IDs has to be created separately using bcm_range_create API as usual. Due to limited resource availability, two groups with same set of Range IDs cannot be created, groups should be unique. Range Group ID could be reused for any field group as its a global resource. After initializing the range_group_config using bcm_range_group_config_t_init API, User need to update required members (as mentioned below) of it before passing it to bcm_range_group_create API.

  • range_bmp : This is a bitmap to specify the Range IDs to be ugrouped together. BCM_RANGE_ID_BMP_XXX macros could be used to add/remove the Range IDs to the bitmap.
  • ports : This is a PBMP to specify for which pipe of XGS pipeline user wants to create this range checker. User can specify a specific pipe or all pipes. User needs to call bcm_port_config_get API to get 'per pipe'/all pipe PBMP. This parameter is mandatory for all range checker types.
Parameters
[in]unitUnit number.
[in,out]range_group_configRange Group Config structure.
Return values
BCM_E_NONERange Module created successfully.
BCM_E_EXISTSRange Group already exists.
BCM_E_UNAVAILFeature not supported.
BCM_E_INITRange module not initialized.
BCM_E_RESOURCERange Group Ids are all allocated
BCM_E_XXXStandard error code.

◆ bcm_range_oper_mode_set()

int bcm_range_oper_mode_set ( int  unit,
bcm_range_oper_mode_t  oper_mode 
)

Set Operational mode for Range Module.

Configure Operational mode of Range Module to either Global or Pipe Local. The operational mode needs to be set before creating Range checkers as otherwise default mode is taken as Global mode. To qualify the range checkers created through new APIs in field module, then both should be in same operational mode.

Parameters
[in]unitUnit number.
[in]oper_modeRange Operational Mode enum value.
Return values
BCM_E_NONEOperation successful.
BCM_E_PARAMInvalid operational mode.
BCM_E_UNITRange module not initialized.
BCM_E_BUSYRanges are already existing. Hence cannot change operational mode.
