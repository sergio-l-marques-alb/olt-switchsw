Multicast Configuration

The following section describes the multicast APIs.

Multicast Overview

This module provides APIs for creating multicast groups for use in packet replication. APIs also exist to add/remove ports to/from the replication list of a multicast group. A multicast group is identified by a unique ID assigned by the SDK (bcm_multicast_t type). The group ID can be used in subsequent APIs to direct traffic to the multicast group (for example, bcm_l2_addr_add and bcm_mpls_vpn_id_create).

The bcm_mcast_* APIs and the bcm_ipmc_* APIs also provide multicast capability for L2 and L3 traffic. These legacy APIs should not be used if the bcm_multicast_* APIs are employed. The multicast module supersedes the previous mcast and IPMC modules.

Note that the multicast module must be used to associate replication lists with features such as VLPS, MAC-in-MAC, Subports, WLAN, layer 2 logical port, and NIV.

Per multicast group controls can be configured using bcm_multicast_control_set and bcm_multicast_control_get APIs.

Multiple Groups Multicast Multiple Groups Related Data types.

  • The bcm_multicast_multi_info_t structure provides the information to create multiple multicast groups.

typedef bcm_multicast_multi_info_s {
uint32 flags; /* BCM_MULTICAST_MULTI_XXX type flags */
int no_of_elements; /* number of elements to be created */
} bcm_multicast_multi_info_t;
Multicast multi Flags
FLAG Meaning
BCM_MULTICAST_MULTI_WITH_ID flag to create multiple multicast groups with a specific base multicast group.
BCM_MULTICAST_MULTI_TYPE_L3 flag to create multiple L3 type multicast groups.
BCM_MULTICAST_MULTI_TYPE_VPLS flag to create multiple VPLS type multicast groups

Multicast Multiple Groups APIs

The multicast multiple groups APIs are used to alloc/free multiple consecutive multicast groups in single API.
