Enum that controls the way the egress queues congetion state will be reported via OOB FC Tx

Enumerator
bcmOOBFCTxReportUCOnly 

OOB FC Tx only reports unicast queues congestion state only.

bcmOOBFCTxReportMCOnly 

OOB FC Tx only reports multicast queues congestion state only.

bcmOOBFCTxReportCos 

OOB FC Tx only reports Cos queues level congestion state.

OOB FC Tx only reports both UC and MC queues congestion state.

◆ bcm_oob_fc_tx_port_mapping_get()

int bcm_oob_fc_tx_port_mapping_get ( int  unit,

int *  port_array,
int *  oob_port_array,
int *  count 
)

Set / Get logical port to OOB port mapping(s)

Set / Get logical port to OOB port mapping(s). Logical/device port to OOB port is strictly one-to-one mapping. On Trident 4 device, it is not allowed to map a logical port to an invalid OOB port(SDKLT only allows to set valid OOB ports), so in previous chips, when user calls bcm_oob_fc_tx_port_mapping_set without specifying logical ports, SDK6 will set OOB port (255) for such logical ports by default, so that user can turn off OOBFC per port. But now on Trident 4, user cannot use this API to implement such functionality, instead port controls bcmPortControlOOBFCTxReportIngEnable/ bcmPortControlOOBFCTxReportEgrEnable are recommended to use to enable/disable OOB reporting per port.

Parameters
[in]unitUnit number.
[in]max_countAllocated number of device port to oob port mappings
[in]port_arraylogical port array
(for "_get") retuned logical port array
[out]oob_port_arrayOOB port array
(for "_get") returned OOB port array
[out]count(for "_set") The number of device port to OOB port mappings.
(for "_get") the number of mappings returned
