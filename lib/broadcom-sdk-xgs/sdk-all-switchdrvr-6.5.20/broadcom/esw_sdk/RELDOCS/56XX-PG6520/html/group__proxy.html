int *  enable 
)

Retrieve per-ingress port proxy server configuration.

This API retrieves the current proxy server configuration on a given HiGig stack port. If proxy_server mode is NOT BCM_PROXY_MODE_SECOND_PASS, get the HiGig lookup or Hybrid lookup configuration on a given HiGig stack port. If proxy_server mode is BCM_PROXY_MODE_SECOND_PASS, get the second pass configuration on a given HiGig stack port. In this mode the flow_type must be identified.

Parameters
[in]unitUnit number.
[in]local_ingress_portIncoming HiGig stack port
[in,out]proxy_server(IN/OUT) Proxy server configuration
[out]enableEnable proxy server on selected HiGig stack port
Return values
BCM_E_XXX

◆ bcm_proxy_server_port_set()

int bcm_proxy_server_port_set ( int  unit,
bcm_gport_t  local_ingress_port,
bcm_proxy_server_t *  proxy_server,
int  enable 
)

Configure per-ingress port proxy server behavior.

This API enables various kinds of lookups on XGS3 devices on behalf of remote legacy or XGS3 devices on a per-ingress port basis. The local ingress port is expected to be a HiGig stack port. The server_gport in the proxy_server structure is the target BCM_GPORT_PROXY (modid,port) destination which indicates that hybrid proxy lookup is required.

bcm_proxy_server_port_set Parameters
Mode Enable parameter Purpose
BCM_PROXY_MODE_HIGIG TRUE Enable HiGig lookup
BCM_PROXY_MODE_HYBRID TRUE Enable Hybrid lookup
BCM_PROXY_MODE_SECOND_PASS TRUE Enable second pass processing
NULL FALSE Disable HiGig lookup or Hybrid lookup
BCM_PROXY_MODE_SECOND_PASS FALSE Disable second pass processing

In the BCM_PROXY_MODE_SECOND_PASS mode, the flow_type must be identified.

Parameters
[in]unitUnit number.
[in]local_ingress_portIncoming HiGig stack port
[in]proxy_serverProxy server configuration
[in]enableEnable proxy server on selected HiGig stack port
Return values
BCM_E_XXX

◆ bcm_proxy_server_t_init()

void bcm_proxy_server_t_init ( bcm_proxy_server_t *  proxy_server)

Initialze a bcm_proxy_server_t structure.

Initializes a proxy server information structure to default values. This function should be used to initialize any proxy server information structure prior to filling it out and passing it to an API function. This ensures that subsequent API releases may add new structure members to the bcm_proxy_server_t structure, and bcm_proxy_server_t_init will initialize the new members to correct default values.

Parameters
[in,out]proxy_server(IN/OUT) Pointer to proxy server structure to initialize
Return values
