◆ bcm_mim_port_add()

int bcm_mim_port_add ( int  unit,
bcm_mim_vpn_t  vpn,
bcm_mim_port_t *  mim_port 
)

Add a MAC-in-MAC port to a VPN.

Add a MAC-in-MAC port to a VPN. Backbone ports are not associated with any VPN. Customers should make use of a Reserved MiM VPN to add all backbone ports within their system. Access and peer ports are added to the broadcast group of the VPN. Backbone ports need to be explicitly added to the broadcast group of a VPN using the bcm_multicast_egress_add API. If the unknown multicast and unknown unicast groups of the VPN are different from the broadcast group, then the bcm_multicast_egress_add API must be used to add ports to them if required. Incoming MAC-in-MAC tunnels are terminated based on the destination MAC and VLAN of the MAC-in-MAC encapsulation. The outer destination MAC must be one of the station MAC addresses. Thus, in addition to creating a backbone port, which is based on the outer source MAC and VLAN, a MAC-in-MAC tunnel termination entry must be created using the bcm_l2_tunnel_add API. The network_group_id member of parameter mim_port is used only for devices supporting multiple split-horizon groups (BCM5645x devices, for example). This is used when the MAC-in-MAC port flags parameter flags = BCM_MIM_PORT_TYPE_PEER or BCM_MIM_PORT_TYPE_ACCESS. The split-horizon group configuration for the given network_group_id can be done using API bcm_switch_network_group_config_set The following table describes the MAC-in-MAC port flags:

◆ bcm_mim_vpn_create()

int bcm_mim_vpn_create ( int  unit,
bcm_mim_vpn_config_t *  info 
)

Create a MAC-in-MAC VPN.

Create a MAC-in-MAC VPN. The bcm_mim_vpn_config_t structure is described below:

typedef struct bcm_mim_vpn_config_s {
uint32 flags; // BCM_MIM_VPN_xxx flags
bcm_mim_vpn_t vpn; // VPN ID
int lookup_id; // I-SID
uint16 match_service_tpid; // Incoming SD tag TPID
bcm_multicast_t broadcast_group; // Broadcast group
bcm_multicast_t unknown_unicast_group; // Unknown unicast group
bcm_multicast_t unknown_multicast_group; // Unknown multicast group
bcm_policer_t policer_id; // Policer ID to be used
bcm_if_t service_encap_id; // Service encap Identifier
uint8 int_pri; // Fixed internal priority
int qos_map_id; // Mapped priority
bcm_vlan_t tunnel_vlan; // B-domain VID
} bcm_mim_vpn_config_t;

The BCM_MIM_VPN_MIM or BCM_MIM_VPN_ELINE flag must be passed in the flags field (see below). A VPN is assigned a unique ID which is returned in the vpn field. The lookup_id is used to specify the 24-bit I-SID value. The multicast group(s) for broadcast and unknown unicast/multicast must first be created using the bcm_multicast_* APIs for vpn type BCM_MIM_VPN_MIM.

MAC-in-MAC VPN Config Flags
Name Purpose
BCM_MIM_VPN_MIM Must be set to specify that the VPN type is MAC-in-MAC.
BCM_MIM_VPN_WITH_ID Used when the application provides a VPN ID.
BCM_MIM_VPN_REPLACE Used when modifying the attributes of an existing VPN.
BCM_MIM_VPN_MATCH_SERVICE_VLAN_TPID Used to specify the TPID to match for SD-tagged packets.
BCM_MIM_VPN_PBB_TE P2P service for Provider Backbone Transport.
BCM_MIM_VPN_BVLAN Creates Backbone VLAN.
BCM_MIM_VPN_SERVICE_ENCAP_WITH_ID Used when the application provides service_encap_id.
BCM_MIM_VPN_QOS_SDTAG_NO_ACTION SDTAG priority control per Mac-in-Mac VPN.
BCM_MIM_VPN_QOS_SDTAG_FIX_PRIORITY SDTAG priority control per Mac-in-Mac VPN, fixed priority.
BCM_MIM_VPN_QOS_SDTAG_MAP_INT_PRIORITY SDTAG priority control per Mac-in-Mac VPN, mapped priority.
BCM_MIM_VPN_QOS_ITAG_FIX_PRIORITY ITAG priority control per Mac-in-Mac VPN, fixed priority.
BCM_MIM_VPN_QOS_ITAG_MAP_SDTAG_PRIORITY ITAG priority control per Mac-in-Mac VPN, mapped priority.
BCM_MIM_VPN_QOS_INGRESS_MAP_FIX_INT_PRIORITY Fixed priority per Mac-in-Mac VPN based on service (VFI) instance.
BCM_MIM_VPN_QOS_INGRESS_MAP_INT_PRIORITY Mapped priority per Mac-in-Mac VPN based on service (VFI) instance.
BCM_MIM_VPN_BVID_DROP_NOMATCH Drop Mac-in-Mac pkt if BVID check fails.
BCM_MIM_VPN_ELINE Must be set to specify that the VPN type is Point-to-Point(E-Line) MAC-in-MAC.
BCM_MIM_VPN_SERVICE_ENCAP_INGRESS_WIDE Create Mac-in-Mac VPN service_encap_id wide mode.
BCM_MIM_VPN_EGRESS_SERVICE_TPID_REPLACE SDTAG TPID control per Mac-in-Mac VPN, replaced TPID.
BCM_MIM_PORT_SERVICE_ENCAP_INGERSS_WIDE Used when the application creates Mac-in-Mac VPN port service_encap_id wide mode.
BCM_MIM_VPN_MATCH_SERVICE_SMAC Match entry include Source B-MAC address.

In a multi-chassis system, same resource may need to be programmed on multiple line cards. (1) In such a scenario, use bcm_mim_vpn_id_create API to obtain a vpn_id(BCM_MIM_VPN_WITH_ID not to be used). (2) BCM_MIM_VPN_WITH_ID along with VPN_ID obtained from step-1 to program same resource in other line cards.

Parameters
[in]unitUnit number.
[in,out]info(IN/OUT) VPN info
