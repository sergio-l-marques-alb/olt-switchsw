◆ bcm_ipmc_add()

int bcm_ipmc_add ( int  unit,
bcm_ipmc_addr_t *  data 
)

Add a new entry to IPMC table.

Adds a new entry to IPMC table. Caller must provide an IPMC group ID that is returned by bcm_multicast_create . On StrataXGS? III devices, multiple IPMC entries can share the same IPMC group ID. If the IPMC entry is associated with a PIM-BIDIR rendezvous point tree, the rp_id member of the bcm_ipmc_addr_t structure should be set to the value returned by bcm_ipmc_rp_create, else rp_id does not need to be set since it's initialized by bcm_ipmc_addr_t_init to BCM_IPMC_RP_ID_INVALID. When BCM_IPMC_REPLACE is set, if the IPMC entry is already present, the entry will be replaced, else new IPMC entry will be created. In BCM5696X, multiple IPMC entries can share the same IPMC group without subject to source port check i.e. multiple IPMC entries need not have same source port in order to share an IPMC group ID. group_l2 field of parameter data is used to provide the ipmc group ID in case of L3 check error on the packet. This API will not do any invalidation on group_l2, it's user's responsibility to maintain the relationship(e.g. ref_count) of group_l2 and IPMC table.

Parameters
[in]unitBCM device number
[in]dataIPMC entry information

Run IPMC table aging.

Performs an aging pass on the IPMC table. All of the entries with the hit bit set have the hit bit cleared. All of the entries without the hit bit set deleted. The caller-supplied age out callback function is called for each deleted ipmc entry. The valid flag is:

  • BCM_IPMC_IP6 - If set, then clear IPv6 entries only. else then clear IPv4 entries only.
Parameters
[in]unitBCM device number
[in]flagsOperation control flags
[in]age_cbCallback function, NULL if none
[in]user_dataUser data to be passed to callback function

◆ bcm_ipmc_counters_get()

int bcm_ipmc_counters_get ( int  unit,
bcm_port_t  port,
bcm_ipmc_counters_t *  counters 
)

Get the combined IP multicast statistics.

Get the IP Multicast statistics for a port. The statistics are returned using the following structure.

typedef struct bcm_ipmc_counters_s {
uint64 rmca; // Received L2 multicast frame
uint64 tmca; // Transmit L2 multicast packet counter
uint64 imbp; // Number of IPMC packets bridged
uint64 imrp; // Number of IPMC packets routed
uint64 rimdr; // Number of IPMC dropped in ingress
uint64 timdr; // Number of IPMC dropped in egress
} bcm_ipmc_counters_t;

This API is equivalent to calling the bcm_stat_get function for the port to retrieve the same statistics. The mapping between statistics and bcm_ipmc_counters_t structure elements is:

BCM IPMC Statistics
Structure Element Statistic
rmca snmpIfHCInMulticastPkts
tmca snmpIfHCOutMulticastPkts
imbp snmpBcmIPMCBridgedPckts
imrp snmpBcmIPMCRoutedPckts
rimdr snmpBcmIPMCInDroppedPckts
timdr snmpBcmIPMCOutDroppedPckts
Parameters
[in]unitUnit number.
[in]portPort to configure
[out]countersIPMC statistics information.

Remove an entry from IPMC table.
