◆ bcm_int_metadata_field_add()

int bcm_int_metadata_field_add ( int  unit,
int  profile_id,
bcm_int_metadata_field_entry_t *  field_entry 
)

Add the metadata field to the metadata profile.

The same field cannot be added into a metadata profile twice. This means if all the parameters in the select and construct structure are exactly the same, it will fail when added the second time. Changing any of these parameters can be treated as a different field. Do not support replace operation for field add. User need to invoke delete and add API in turn. SDK will not check the overlap issue.

◆ bcm_int_metadata_profile_create()

int bcm_int_metadata_profile_create ( int  unit,
uint32  options,
int *  profile_id 
)

Create the metadata profile.

First create the metadata profile, and then add the field entry into the profile to add the field. It supports max 12 16bits field selectors and 12 32bits field selectors for one profile. User can invoke API bcm_int_metadata_field_selct_info_get to obtain the select information for the given field. field_start/field_size should be set in reasonable range according to the select information while adding the field by API bcm_int_metadata_field_add. When field start is set with non-zero value to select partial data of the field, or the field is less than 16 bits and the offset in the chunk is not zero, user can shift left the field by setting field shift.

Parameters
[in]unitUnit number.
[in]optionsCreate options
[in,out]profile_idMetadata profile ID

uint32  options,

bcm_int_opaque_data_profile_t *  profile 
)

Create the opaque data profile for INT metadata construction.

If the pointer from L3 routing and FP are not configured, the table will be indexed with egress port number. When user want to configure the profile on the port, they need to create the profile with ID (ID equals the port number). In this mix mode, auto assignment is not recommended for creating other profiles indexed by L3 routing and FP. Because user should avoid to occupy the profile entry of the port by themselves. SDK will not check if the same profiles are created meanwhile. User should guarantee it by themselves.
