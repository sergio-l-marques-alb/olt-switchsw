Port Extension Management

BCM56640 supports the IEEE 802.1Qbh (Bridge Port Extension) functionality. The BCM Port Extension APIs enable application developers to manage this feature.

Port Extension Overview

In today's data center, there are multiple layers of L2 switches between virtual machines and top-of-the-rack switches, introduced by hypervisors, converged network adapters, and blade switches. These intermediate switches complicate network management as the server is no longer directly connected to a port on the top-of-the-rack switch. IEEE 802.1Qbh addresses this situation by defining two new network elements, Port Extender and Controlling Bridge. The intermediate switches serve as Port Extenders, which forward frames between virtual machines and the Controlling Bridge. The top-of-the-rack switch serves as the Controlling Bridge, the central point for making forwarding decisions and enforcing policy. The frames flowing between Port Extender and Controlling Bridge carry an ETAG. In the upstream direction, ETAG's Extended Port VLAN ID field identifies the source Port Extender port. In the downstream direction, the same field identifies either the destination Port Extender port or the downstream multicast tree.

Port Extension Data Types

  • The bcm_extender_port_t structure describes a Port Extender port logically attached to a Controlling Bridge.
typedef struct bcm_extender_port_s {
uint32 flags; /* BCM_EXTENDER_PORT_xxx. */
bcm_gport_t extender_port_id; /* GPORT identifier. */
bcm_gport_t port; /* Physical port / trunk */
uint16 extended_port_vid; /* Extender port VID. */
bcm_vlan_t match_vlan; /* Outer VLAN ID to match. */
bcm_extender_pcp_de_select_t pcp_de_select; /* Selection of PCP and DE fields for
egress ETAG. */
int qos_map_id; /* Qos map id for egress etag mapping profile. */
uint8 pcp; /* Default PCP field of ETAG. */
uint8 de; /* Default DE field of ETAG. */
bcm_if_t encap_id; /* Encap identifier. */
bcm_gport_t failover_port_id; /* Failover id. */
bcm_failover_t failover_id; /* Failover object index. */
bcm_failover_t ingress_failover_id; /* 1+1 protection. */
bcm_multicast_t failover_mc_group; /* MC group for 1+1 scheme failover. */
int inlif_counting_profile; /* In LIF counting profile */
int outlif_counting_profile; /* Out LIF counting profile */
} bcm_extender_port_t;
BCM_EXTENDER_PORT flags
Name Purpose
BCM_EXTENDER_PORT_WITH_ID Create Port Extender port with specified GPORT ID
BCM_EXTENDER_PORT_REPLACE Replace existing Port Extender port
BCM_EXTENDER_PORT_MULTICAST Create a downstream multicast tree
BCM_EXTENDER_PORT_ENCAP_WITH_ID Use the specified encap id to allocate and setup encapsulation.
BCM_EXTENDER_PORT_EGRESS_TRANSPARENT Use the flag to transmit packets transparently out of this extender port. It means ETAG of the packets out of this extender port will not be modified.
BCM_EXTENDER_PORT_DROP Drop egress packets belonging to the extender port
BCM_EXTENDER_PORT_ID_ASSIGN_DISABLE Matched traffic is not assigned to the extender port
BCM_EXTENDER_PORT_INGRESS_WIDE Creates encap id wide mode
typedef enum bcm_extender_pcp_de_select_e {
BCM_EXTENDER_PCP_DE_SELECT_OUTER_TAG, /* Derive ETAG's PCP and DE fields from outer VLAN tag */
BCM_EXTENDER_PCP_DE_SELECT_INNER_TAG, /* Derive ETAG's PCP and DE fields from inner VLAN tag */
BCM_EXTENDER_PCP_DE_SELECT_DEFAULT, /* Derive ETAG's PCP and DE fields from default values */
BCM_EXTENDER_PCP_DE_SELECT_PHB /* Derive ETAG's PCP and DE fields from per-hop behavior */
} bcm_extender_pcp_de_select_t;
  • The bcm_extender_forward_t structure describes a downstream forwarding entry in a Port Extender.
typedef struct bcm_extender_forward_s {
uint32 flags; /* BCM_EXTENDER_FORWARD_xxx. */
uint16 name_space; /* extender Namespace. */
uint16 extended_port_vid; /* Extended port VID. */
bcm_gport_t dest_port; /* Destination Gport. */
bcm_multicast_t dest_multicast; /* Destination Multicast Group. */
bcm_gport_t extender_forward_id; /* Downstream forwarding id. */
uint32 class_id; /* Class id. */
} bcm_extender_forward_t;
BCM_EXTENDER_FORWARD flags
Name Purpose
BCM_EXTENDER_FORWARD_REPLACE Replace existing forwarding entry
BCM_EXTENDER_FORWARD_MULTICAST Destination is a multicast group
BCM_EXTENDER_FORWARD_UPSTREAM_ONLY If set, the API configures upstream (ECID registration) information only.
BCM_EXTENDER_FORWARD_DOWNSTREAM_ONLY If set, the API configures downstream information only.
BCM_EXTENDER_FORWARD_WITH_ID Set extender_forward_id value. Used in case of downstream only.
BCM_EXTENDER_FORWARD_COPY_TO_CPU Copy to CPU for downstream
  • The bcm_extender_encap_t structure describes Port PCID encapsulation entry in a Port Extender.
typedef struct bcm_extender_encap_s {
uint32 flags; /* BCM_EXTENDER_ENCAP_xxx. */
uint16 extended_port_vid; /* Extended port VID - PCID value. */
bcm_gport_t extender_encap_id; /* Extender encapsulation id. */
} bcm_extender_encap_t;
BCM_EXTENDER_ENCAP flags
Name Purpose
BCM_EXTENDER_ENCAP_WITH_ID Add using the specified extender_encap_id value
BCM_EXTENDER_ENCAP_REPLACE Replace existing extender_encap_id entry
