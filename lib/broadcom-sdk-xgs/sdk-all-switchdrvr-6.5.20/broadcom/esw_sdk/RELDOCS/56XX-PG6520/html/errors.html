Error Codes

With few exceptions, all BCM routines return an integer indicating the success or error status of the call. All error codes (except BCM_E_NONE) are negative and are one of the values given in error_codes . Success is usually indicated by the return value of BCM_E_NONE, which is defined to be zero. Some routines may also return other success indications as positive numbers. Any such deviations from error_codes will be described in the individual API routine descriptions.

In the API descriptions, an arbitrary error return code is specified by writing BCM_E_XXX. This means that the routine will return some error code listed in error_codes .

Routines may also propagate error indications from lower-level driver and utility routines. The error codes thus propagated are compatible with these BCM error codes, and can be treated identically.

Access to the error definitions is through the include file <bcm/error.h>.

Checking for the failure of an API call can be done by testing if its return value is negative or by using BCM_FAILURE macro:

int rv;
rv = bcm_port_speed_set(int unit, bcm_port_t port, int speed);
if (rv < 0) {
/* failure */
}
if (BCM_FAILURE(rv)) {
/* failure */
}

Error Codes
Error Code String Value Description
BCM_E_NONE Ok The routine completed successfully.
BCM_E_INTERNAL Internal error Some internal operations failed in an unexpected way. This usually indicates that an access to some hardware resource failed.
BCM_E_MEMORY Out of memory Dynamically allocated memory was not available to complete the operation.
BCM_E_UNIT Invalid unit The first argument of any BCM API routine is a unit number. This error occurs if that number does not refer to a currently attached unit.
BCM_E_PARAM Invalid parameter A parameter to a BCM API routines was invalid. A null pointer value may have been passed to the routine, or an integer parameter may be outside of its allowed range.
BCM_E_EMPTY Table empty The routine expected a table not to be empty.
BCM_E_FULL Table full The routine could not add an entry to the table
BCM_E_NOT_FOUND Entry not found The specific entry in a table could not be found.
BCM_E_EXISTS Entry exists The specific entry was already in a table
BCM_E_TIMEOUT Operation timed out The routine started an operation that did not complete.
BCM_E_BUSY Operation still running The routine tried to start an operation, but a previous operation was still running.
BCM_E_FAIL Operation failed The routine attempted an operation that could not be completed. This may be because of a hardware failure or a configuration failure.
BCM_E_DISABLED Operation disabled The routine could not complete because a required feature was temporarily disabled. This often relates to auto-negotiation.
BCM_E_BADID Invalid identifier An identifier was not valid.
BCM_E_RESOURCE No resources for operation The routine needed more hardware resources than were available.
BCM_E_CONFIG Invalid configuration The hardware does not support the requested configuration.
BCM_E_UNAVAIL Feature unavailable The hardware does not support the requested routine.
BCM_E_INIT Feature not initialized The routine used a part of the API that was not yet initialized.
BCM_E_PORT Invalid port A port parameter was given that is not valid.
BCM_E_IO I/O access failed. A low-level register or memory access failed.
BCM_E_ACCESS Access method not permitted. Access method not permitted. Typically returned if attempting to write to a read-only object.
BCM_E_HANDLER No handler. No handler exists to perform the hardware access associated with an operation on a software object.
BCM_E_PARTIAL Partial success. The operation was only partially completed, and this could potentially leave the system in an unexpected state.
API Brief
bcm_errmsg() Convert an error code into a printable string.
