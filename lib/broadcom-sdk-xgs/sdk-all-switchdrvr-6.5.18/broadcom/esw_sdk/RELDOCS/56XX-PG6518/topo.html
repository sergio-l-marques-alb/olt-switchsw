Topology Mapping Lookup

When multiple devices coexist in a system, their interconnections form a topology. Packets flowing through the system are often directed by a fabric module ID, so called because switch fabrics use this identifier to determine the path taken by the packet.

Some API routines need to determine the path to take to go from one device to a given fabric module ID. The Topology Mapping APIs provide this functionality. The mechanism for this may be as simple as indicating a single port (because all external modules are reached through that one port). A switch fabric, however, would require a table lookup to see which port should be used.

Each board type needs a specific function that does the appropriate lookup from either hardware or software tables based on the type of unit.

If no mapping function is set, then a default routine that can handle many single board configurations is used.

These APIs may disappear (or become internal) in future releases.

API Brief
bcm_topo_map_f() Topology mapping lookup function prototype.
bcm_topo_map_set(bcm_topo_map_f _map) Set or Get Topology mapping lookup function.
bcm_topo_map_get(bcm_topo_map_f* _map) Set or Get Topology mapping lookup function.
bcm_topo_port_get(int unit, int dest_modid, bcm_port_t* exit_port) Lookup a path to a destination module.
