Fiber Channel Over Ethernet

Fiber Channel Over Ethernet Overview

Fibre Channel over Ethernet (FCoE) is an industry standard that allows Fibre Channel to be carried over Ethernet links. It leverages data center bridging (DCB) enhancements that provide lossless and reliable Ethernet. With FCoE, network (IP) and storage (SAN) can be consolidated on a single converged network.

The following functionality is provided by FCoE API

Routing: bcm_fcoe_route_* API manages the functionality to route the FCOE packets based on the FC header. Supports the ability to route based on host routes (fully specified D_ID/S_ID), domain routes and prefix routes.

Zoning: bcm_fcoe_zone_* API provides the ability add zoning checks to FCoE flows. Zone checks have actions that are configurable, see FCOE_zone_actions_table for a list of them. In addition there are switch controls (see bcm_switches) to control what actions to take for packets that either miss or fail zone checks.

Vsan Management: bcm_fcoe_vsan_* API provides the ability to manage VSANs. On a per VSAN basis, there are checks (see fcoe_vsan_controls_table) that can be enabled along with the ability to associate an FC-MAP.

Port controls: Port specific FCoE controls are provided using bcmPortControlFcoe* API (see PORT_CONTROL_v).

Switch controls: FCoE controls which are global to the device are provided using bcmSwitchFcoe* API (see bcm_switches).

Filtering and Snooping: FCOE/FC header fields are available at the Ingress and Egress stages of CAP (Content Aware Processor). This allows full control to program rules to filter and snoop. See Field Processor for further details.

FCoE translation: bcm_fcoe_vsan_translate_* API provides the ability to control vlan and VFT tags. In addition, VSAN/VFT tagging can affect the VLAN tag. See new FCoE specific bcm_vlan_action_t actions that have been added.

FCoE interface: bcm_fcoe_intf_* API provides the mechanism to set the FCOE specific properties for egress interface created using bcm_l3_intf_create.

QOS mapping: New BCM_QOS_MAP_VFT and BCM_QOS_MAP_VSAN priority maps have been. BCM_QOS_MAP_VFT controls how VFT tag priority maps to internal priority. BCM_QOS_MAP_VSAN controls how the internal priority and congestion map to outgoing packet priority.

  • The bcm_fcoe_route_t struct
typedef struct bcm_fcoe_route_s {
uint32 flags; /* BCM_FCOE_* flags */
bcm_vrf_t vrf; /* Virtual router instance */
bcm_fcoe_nport_id_t nport_id; /* Destination/Source FC Nport ID */
bcm_fcoe_nport_id_t nport_id_mask; /* Mask used for Domain/prefix routes */
bcm_cos_t pri; /* New priority in packet */
bcm_if_t intf; /* L3 intf associated with this address */
bcm_mac_t nexthop_mac; /* Next-hop mac address */
bcm_gport_t port; /* Port */
int lookup_class; /* Classification lookup class ID */
bcm_if_t encap_id; /* encapsulation index */
} bcm_fcoe_route_t;
