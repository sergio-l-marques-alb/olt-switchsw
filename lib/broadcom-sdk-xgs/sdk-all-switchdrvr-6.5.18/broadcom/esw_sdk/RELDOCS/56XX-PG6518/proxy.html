Proxy Services

The Proxy Service enables a System (chassis or set of stackable switches) to support new or advanced StrataXGS III data path features on behalf of a single or multiple legacy or StrataXGS III devices that may also be part of the same System.

The silicon design of StrataXGS III takes into consideration end customer investment protection by allowing an upgrade path from current silicon to exercise new features. This Proxy feature allows existing StrataXGS I and StrataXGS II customers to upgrade the feature set of an existing system by adding an StrataXGS III element into a stackable system or a chassis line card.

Proxy clients are devices where packets ingressed originally. Packets are then redirected to a port on remote StrataXGS III device, which can be thought of as a Proxy Server device. The port on Proxy Server device could either be a HG port or a GbE/XE port, depending on the capability of the Server StrataXGS III device. The API provides for different modes of Proxy Server configuration. On the StrataXGS III Server device, packets will undergo processing in various stages just like a packet that comes in on a customer (GbE/XE) port.

The BCM Proxy APIs allow applications to efficiently utilize this cost effective feature, with minimal knowledge of the switch chip internals.

Data Types Defined for Proxy APIs

  • bcm_proxy_mode_t The type bcm_proxy_mode_t is an enumeration of the various possible Proxy server modes that an StrataXGS III device (and/or port) can be configured to be in. Applications can choose any of the below Proxy Server modes depending on the StrataXGS III device capabilities.
typedef enum bcm_proxy_mode_s {
BCM_PROXY_MODE_NONE,
BCM_PROXY_MODE_LOOPBACK, /* Loopback on GE/XE port */
BCM_PROXY_MODE_HIGIG, /* HG lookup capability */
BCM_PROXY_MODE_HYBRID, /* HG hybrid capability */
BCM_PROXY_MODE_SECOND_PASS /* Second pass processing */
} bcm_proxy_mode_t;
  • bcm_proxy_proto_type_t The type bcm_proxy_proto_type_t is an enumeration of various possible traffic types used to classify packets that need to be redirected.

BCM_PROXY_PROTO_IP6_ALL, /* All IPv6 packets */
BCM_PROXY_PROTO_IP6_UCAST, /* IPv6 Unicast packets only */
BCM_PROXY_PROTO_IP6_MCAST, /* IPv6 Multicast packets only */
BCM_PROXY_PROTO_MPLS_UCAST, /* MPLS Unicast packets only */
BCM_PROXY_PROTO_MPLS_MCAST, /* MPLS Multicast packets only */
BCM_PROXY_PROTO_MPLS_ALL, /* All MPLS packets */
BCM_PROXY_PROTO_IP6_IN_IP4, /* IPv6-in-IPv4 (proto ) pkts */

  • bcm_proxy_server_t contains the information to specify a proxy server destination on a per-ingress port basis. This port-specific model only supports the BCM_PROXY_MODE_HIGIG and BCM_PROXY_MODE_HYBRID modes of operation. The flags selections are described in the table below.
typedef struct bcm_proxy_server_s {
uint32 flags; /* BCM_PROXY_SERVER_xxx. */
bcm_gport_t server_gport; /* Proxy GPORT identifier. */
bcm_gport_t server_gport_mask; /* Proxy GPORT Mask identifier to selectively mask DST_PORT and DST_MODID for signature match. */
bcm_proxy_mode_t mode; /* Selected proxy server operational mode. */
bcm_proxy_second_pass_flow_type_t flow_type; /* Flow type identifier. */
} bcm_proxy_server_t;

Proxy APIs

API Brief
bcm_proxy_init(int unit) Initialize the BCM Proxy subsystem.
bcm_proxy_cleanup(int unit) Detach the Proxy software module.
bcm_proxy_client_set(int unit, bcm_port_t client_port, bcm_proxy_proto_type_t proto_type, bcm_module_t server_modid, bcm_port_t server_port, int enable) Configure a proxy client for the given unit and port. The server module ID and port must be valid.
bcm_proxy_server_set(int unit, bcm_port_t server_port, bcm_proxy_mode_t mode, int enable) This API enables various kinds of lookups on StrataXGS III device on behalf of remote legacy or StrataXGS III devices.
bcm_proxy_server_get(int unit, bcm_port_t server_port, bcm_proxy_mode_t mode, int* enable) Get the lookup status (enabled/disabled) of StrataXGS III Proxy server device, given the Server mode.
bcm_proxy_server_t_init(bcm_proxy_server_t* proxy_server) Initialze a bcm_proxy_server_t structure.
bcm_proxy_server_port_set(int unit, bcm_gport_t local_ingress_port, bcm_proxy_server_t* proxy_server, int enable) Configure per-ingress port proxy server behavior.
bcm_proxy_server_port_get(int unit, bcm_gport_t local_ingress_port, bcm_proxy_server_t* proxy_server, int* enable) Retrieve per-ingress port proxy server configuration

IPv6 Multicast in a Stackable containing StrataXGS II/legacy devices

IPv6 Multicast can be enabled in a Stackable/Chassis system, that contains a StrataXGS II or legacy device that does not understand IPv6.

Customer IPv6 Multicast traffic may ingress on StrataXGS II or legacy device. Using Proxy client API bcm_proxy_client_set, packets can be redirected to an StrataXGS III device. IPv6 Multicast lookup is done on the StrataXGS III device. If the desired egress ports reside on another device (different from where IPv6 MC lookup was done) which may be an StrataXGS II or legacy device, then the L3 bitmap of the L3_IPMC entry (on StrataXGS III device) must have HG port that connects to the StrataXGS II or legacy device.

Further, the StrataXGS III device (where IPv6 MC lookup happens) must also be programmed to perform IPMC replication on the HG port, as many times as the number of remote egress ports (on StrataXGS II or legacy devices).

See bcm_ipmc_add and bcm_ipmc_remove for details.

Proxy HiGig Proxy

Customer Unicast and Multicast traffic may ingress on BCM56850 device and may need to be forwarded to HiGig Proxy device. Using bcm_proxy_egress_create/delete/get APIs, customer may create Proxy-nexthop to redirect the packets to HiGig Proxy device.

It is important always to initialize an Proxy egress structure using the bcm_proxy_egress_t_init function prior to passing it to an API. The gport parameter contains the Module-ID and Port-ID of the Proxy device.
