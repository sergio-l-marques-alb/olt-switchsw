Policer Group Modes
Mode Description
bcmPolicerGroupModeSingle Default mode where a single policer is used for all traffic types
bcmPolicerGroupModeTrafficType A dedicated policer per traffic type unicast, multicast, and flood.
bcmPolicerGroupModeDlfAll A pair of policers where the base policer is used for dlf and the other policer is used for all traffic types
bcmPolicerGroupModeDlfIntPri N+1 policers where the base policer is used for dlf and next N are used one per Cos level
bcmPolicerGroupModeTyped A dedicated policer for unknown unicast, known unicast, multicast, broadcast
bcmPolicerGroupModeTypedAll A dedicated policer for unknown unicast, known unicast, multicast, broadcast and one for all traffic (not already policed)
bcmPolicerGroupModeTypedIntPri A dedicated policer for unknown unicast, known unicast, multicast, broadcast and N internal priority policers for traffic (not already policed)
bcmPolicerGroupModeSingleWithControl A policer pair, where the base policer is used for control traffic and next policer is used for all data traffic types
bcmPolicerGroupModeTrafficTypeWithControl A set of policers, where the base policer is used for control traffic, and the remaining are data policers, one per traffic type unicast, multicast, and flood.
bcmPolicerGroupModeDlfAllWithControl A set of 3 policers, where the base policer is used for control traffic, the next is for dlf, and a last one for all traffic types
bcmPolicerGroupModeDlfIntPriWithControl A set of N+2 policers where the base policer is used for control traffic, the next is for dlf, and the remaining N are used one per Cos level
bcmPolicerGroupModeTypedWithControl A set of policers, where the base is for control traffic and remaining are for data Each data policer is a dedicated policer for unknown unicast, known unicast, multicast, broadcast
bcmPolicerGroupModeTypedAllWithControl A set of policers, where the base is for control traffic and remaining are for data. Each data policer is a dedicated policer for unknown unicast, known unicast, multicast, broadcast and one for all data traffic (not already policed)
bcmPolicerGroupModeTypedIntPriWithControl A set of policers, where the base is for control traffic and remaining are for data. Each data policer is a dedicated policer for unknown unicast, known unicast, multicast, broadcast and N internal priority policers for traffic (not already policed)
bcmPolicerGroupModeDot1P A set of 8 policers selected based on outer Vlan priority
bcmPolicerGroupModeIntPri A set of 16 policers based on internal priority
bcmPolicerGroupModeShortIntPri A set of 8 policers based on internal priority
bcmPolicerGroupModeIntPriCng A set of 64 policers based on Internal priority and CNG
bcmPolicerGroupModeSvpType A set of 2 policers based on SVP type. For BCM5645x devices, 8 policers are created.
bcmPolicerGroupModeDscp A set of 64 policers based on DSCP bits
bcmPolicerGroupModeCascade User specified set of policers(max 8) wherein excess bandwidth can overflow from High priority to low priority.
bcmPolicerGroupModeCascadeWithCoupling User specified set of policers(max 4) wherein excess bandwidth can overflow from High priority to low priority bucket and from lowest priority green bucket to highest priority yellow.
bcmPolicerGroupModeInnerDot1P A set of 8 policers selected based on inner Vlan priority
bcmPolicerGroupModeEgressSingle A dedicated policer for all egress traffic
bcmPolicerGroupModeEgressIntPri A dedicated policer for egress traffic for each cos level
bcmPolicerGroupModeIntPriCascade A set of policers(max 8) selected based on internal priority, wherein excess bandwidth can overflow from high priority bucket to low priority bucket
bcmPolicerGroupModeIntPriCascadeWithCoupling A set of policers(max 4) selected based on internal priority, wherein excess bandwidth can overflow from high priority bucket to low priority bucket, and from low priority green bucket to high priority yellow
bcmPolicerGroupModePreemptable A set of 2 policers based on whether the packet is preemptable or not
bcmPolicerGroupModeCascadeDroppedPktExclude A set of policers(max 8) selected based on internal priority, wherein excess bandwidth can overflow from high priority bucket to low priority bucket and packets marked as dropped will be excluded from metering.
Policer Flags
Name Purpose
BCM_POLICER_WITH_ID Specify a specific ID to use
BCM_POLICER_REPLACE Replace the existing configuration
BCM_POLICER_COLOR_BLIND Policer is color blind
BCM_POLICER_DROP_RED If marked Red, take drop action
BCM_POLICER_COLOR_MERGE_OR Logical Or Hierarchical policers results.
BCM_POLICER_MODE_BYTES Policer Metering is based on bytes.
BCM_POLICER_MODE_PACKETS Policer Metering is based on packets.
BCM_POLICER_MAX_CKBITS_SEC_VALID Policer max_ckbits_sec is valid.
BCM_POLICER_MAX_PKBITS_SEC_VALID Policer max_pkbits_sec is valid.
BCM_POLICER_MACRO Allocate Policer from group B (default is group A).
BCM_POLICER_EGRESS Allocate Policer for egress usage.
BCM_POLICER_REGEX Allocate Policer for Regex usage.
BCM_POLICER_REPLACE_SHARED Replace the profile rather than the configuration.
BCM_POLICER_MIXED_MICRO_MACRO Micro Policer not associated to Hierarchy/Macro Envelope.
BCM_POLICER_BW_SHARING_GROUP_START Policer corresponds to the start of bandwidth sharing group(cascade chain).
BCM_POLICER_BW_SHARING_GROUP_END Policer corresponds to the end of bandwidth sharing group(cascade chain).
BCM_POLICER_WITH_POOL_ID Policer needs to be created with the given PoolId
BCM_POLICER_AS_PACKET_COUNTER Policer needs to be created as packet counter.
BCM_POLICER_PKT_ADJ_HEADER_TRUNCATE Indicates that this Policer needs to be considerate of header truncate adjustment.
BCM_POLICER_APPLY_GLOBAL_POOL_ID Indicates that this Policer needs to be allocated from the given global meter pool id. Using this flag, user can manage the pool Id allocation on its own. It is recommended to use this flag for all the policers configured in the system in order to maintain the consistency. Policer creation without this flag allocates pool id internally which may conflict with the policer created with pool id.

API Brief
bcm_policer_create(int unit, bcm_policer_config_t* pol_cfg, bcm_policer_t* policer_id) Create a policer entry.
bcm_policer_group_create(int unit, bcm_policer_group_mode_t mode, bcm_policer_t* policer_id, int* npolicers) Allocate a block of policer entries. bcm_policer_set must be called to setup the individual policers.
bcm_policer_aggregate_group_create(int unit, bcm_policer_aggregate_group_info_t* info, bcm_policer_t* policer_id, int* npolicers) Allocate a block of policer entries. bcm_policer_set must be called to setup the individual policers.
bcm_policer_group_create_with_map(int unit, bcm_policer_group_mode_t mode, bcm_policer_map_t* offset_map, bcm_policer_t* policer_id, int* num_policers) Allocate a block of policer entries with flexible mapping. bcm_policer_set must be called to setup the individual policers.
bcm_policer_envelop_create(int unit, uint32 flag, bcm_policer_t macro_flow_policer_id, bcm_policer_t* policer_id) Allocate a micro or a macro meter policer.
bcm_policer_envelop_group_create(int unit, uint32 flag, bcm_policer_group_mode_t mode, bcm_policer_t macro_flow_policer_id, bcm_policer_t* policer_id, int* npolicers) Allocate a group of micro or a macro meter policer. bcm_policer_set must be called to setup the individual policers.
bcm_policer_destroy(int unit, bcm_policer_t policer_id) Destroy a policer entry.
bcm_policer_destroy_all(int unit) Destroy all policer entries.
bcm_policer_set(int unit, bcm_policer_t policer_id, bcm_policer_config_t* pol_cfg) Set the config settings for a policer entry.
bcm_policer_get(int unit, bcm_policer_t policer_id, bcm_policer_config_t* pol_cfg) Get the config settings for a policer entry.
bcm_policer_packet_counter_get32(int unit, bcm_policer_t policer_id, uint32* count) Get the current bucket count for a policer entry.
bcm_policer_packet_reset_counter_get32(int unit, bcm_policer_t policer_id, uint32* count) Reset the bucket count to initial value for a policer entry.
bcm_policer_traverse(int unit, bcm_policer_traverse_cb cb, void* user_data) Traverse the existing policers and call the user-supplied callback function for each one.
bcm_policer_stat_enable_get(int unit, bcm_policer_t policer_id, int* enable) Checks if statistics collection is enabled on the specified policer ID.
bcm_policer_stat_enable_set(int unit, bcm_policer_t policer_id, int enable) Enables statistics collection on the specified policer ID.
bcm_policer_stat_get(int unit, bcm_policer_t policer_id, bcm_cos_t int_pri, bcm_policer_stat_t stat, uint64* value) Gets the specified stat.
bcm_policer_stat_get32(int unit, bcm_policer_t policer_id, bcm_cos_t int_pri, bcm_policer_stat_t stat, uint32* value) 32-bit version of stat get API
bcm_policer_stat_set(int unit, bcm_policer_t policer_id, bcm_cos_t int_pri, bcm_policer_stat_t stat, uint64 value) Sets the specified stat.
bcm_policer_stat_set32(int unit, bcm_policer_t policer_id, bcm_cos_t int_pri, bcm_policer_stat_t stat, uint32 value) 32-bit version of stat set API
