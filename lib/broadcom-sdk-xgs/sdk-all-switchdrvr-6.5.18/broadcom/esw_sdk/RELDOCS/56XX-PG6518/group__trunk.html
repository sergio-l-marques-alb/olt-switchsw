◆ bcm_trunk_create()

int bcm_trunk_create ( int  unit,
uint32  flags,
bcm_trunk_t *  tid 
)

Create the software data structure for a new trunk, using caller-specified or next available trunk ID.

To create a front-panel trunk group, the BCM_TRUNK_FLAG_WITH_ID flag may be set. If the BCM_TRUNK_FLAG_WITH_ID flag is set, the caller-specified trunk ID must be within the range specified by the members trunk_id_min and trunk_id_max of the bcm_trunk_chip_info_t structure, returned by bcm_trunk_chip_info_get . If the BCM_TRUNK_FLAG_WITH_ID flag is not set, this API will return the next available front-panel trunk ID. To create a fabric trunk group, the BCM_TRUNK_FLAG_WITH_ID flag must be set. The user-specified fabric trunk ID must be within the range specified by the members trunk_fabric_id_min and trunk_fabric_id_max of the bcm_trunk_chip_info_t structure. To create a virtual port trunk group, the BCM_TRUNK_FLAG_WITH_ID flag may be set. If the BCM_TRUNK_FLAG_WITH_ID flag is set, the caller-specified trunk ID must be within the range specified by the members trunk_id_min and trunk_id_max of the bcm_trunk_chip_info_t structure, returned by bcm_trunk_chip_info_get . If the BCM_TRUNK_FLAG_WITH_ID flag is not set and the BCM_TRUNK_FLAG_VP is set, the virtual port trunk group will be created with the next available trunk ID. The maximum number of virtual port trunk groups is (trunk_id_max - trunk_id_min + 1). If the SOC property spn_MAX_VP_LAGS is set, it will be equal to the SOC property value - 1. Otherwise, the default value is the maximum number supported by the device. This function does not update any hardware tables. To finish trunk setting, call bcm_trunk_set .

Parameters
[in,out]unitBCM device number
[in,out]flagsBCM_TRUNK_FLAG_*
[in,out]tid(IN/OUT) Pointer to caller-specified trunk ID if BCM_TRUNK_FLAG_WITH_ID is set, else pointer to next available trunk ID.

This function configures the distribution details of the trunk hashing algorithm. For packets on the selected ingress port and trunk ID, the destination trunk port selected by the hash calculation is determined by the given size and weights parameters. The size parameter determines the range of values to which the hash calculation is mapped. The weights parameters give a relative weighting of the ports within the trunk group. The weights array is indexed by the device port number, not the number of the port within the trunk.
