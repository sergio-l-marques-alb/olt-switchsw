Create a fabric qsel.

Creates a fabric qsel, as specified by the flags, whose minimum valid queue index is specified by base, and whose maximum valid queue index is (base + count - 1). If the WITH_ID flag is specified, qsel_id is an IN parameter; otherwise it is an OUT parameter. Note that base and count do not specify literal addresses in any table. They specify the valid range of values pulled from the packet for use as queue index by an action. For example, if a qsel is used by an action that can only generate values 0x0100 through 0x0FFF for the index, the qsel should be created with base=0x0100 and count=0x0F00; in this case, index 0x0100 would specify the lowest entry in the qsel, and index 0x0FFF would specify the highest. The qsel may be located anywhere in available tables as appropriate, and both the APIs and actions will be presented the same index properties.
