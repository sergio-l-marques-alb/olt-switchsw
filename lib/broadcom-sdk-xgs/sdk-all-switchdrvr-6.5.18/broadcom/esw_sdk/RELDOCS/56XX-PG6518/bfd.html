Bidirectional Forwarding Detection

This feature is included in the default FEATURE_LIST for devices that supports it.

It requires the BFD application to be available in the uKernel loaded in the uController.

In addition, BFD requires a CPU COSQ. A queue must be available to use for the uController where the BFD application is running. All queues are assigned by default to the host CPU, so it is necessary to define the right number of queues to be shared among the host CPU and the uControllers.

Queues can be assigned to the host CPU and uControllers with the SOC properties num_queues_pci, num_queues_uc0, num_queues_uc1.

By default, BFD chooses the highest available queue for the uController it is running on. A specific queue can be assigned to BFD with the SOC property bfd_cosq.

For devices which support them, the BFD APIs provide methods for the control and configuration of BFD sessions. The following BFD features are supported and configurable per session:

  • Active or Passive Mode initialization
  • Asynchronous or Demand Mode operation
  • Authentication (Simple Password, SHA1, or none)

Callback mechanisms are provided for notifying the application of BFD-related events, such as change of session state.

BFD Data Types

  • bcm_bfd_tunnel_type_t is an enumeration of BFD encapsulations.
  • bcm_bfd_endpoint_t is an identifier used to refer to a BFD endpoint.
  • bcm_bfd_endpoint_info_t is a structure containing information about the local and remote endpoints of a BFD session.
  • bcm_bfd_auth_simple_password_t is a structure containing information about a simple password authentication entry
  • bcm_bfd_auth_sha1_t is a structure containing information about a SHA1 authentication entry.
  • bcm_bfd_counts_t is a structure containing counter information for a BFD endpoint.
  • bcm_bfd_state_t is an enumeration of the states found in BFD control packets.
  • bcm_bfd_diag_code_t is an enumeration of the diagnostic codes found in BFD control packets.
  • bcm_bfd_auth_t is an enumeration of BFD authentication types.

The following type is used for supplying function pointers to callbacks for event handling.

typedef int (*bcm_bfd_event_cb)(
int unit,
uint32 flags,
bcm_bfd_events_type_t events,
bcm_bfd_endpoint_t endpoint,
void *user_data);

BFD Endpoint Info

The BFD endpoint info structure contains information on both the local and remote ends of the BFD session.

uint32 flags; /* control flags */

bcm_bfd_auth_type_t auth; /* authentication type */

uint32 rx_auth_seq; /* rx authentication sequence ID */
uint32 remote_flags; /* remote flags */
bcm_bfd_state_t remote_state; /* remote session state */
uint32 remote_discr; /* remote discriminator */
bcm_bfd_diag_code_t remote_diag; /* remote diagnostic code */
uint32 remote_min_tx; /* desired remote min tx interval */
uint32 remote_min_rx; /* required remote rx interval */
uint32 remote_min_echo; /* remote minimum echo interval */
uint8 remote_detect_mult; /* remote detection interval multiplier */
int sampling_ratio; /* 0 - No packets sampled to the CPU. 1-8 - Count of packets (with events) that need to arrive before one is sampled to the CPU. */
uint8 loc_clear_threshold; /* Number of packets required to reset
the Loss-of-Continuity status per
endpoint. */

check is performed, positive values
indicate the amount of MSBs to be

REMOTE MEP-ID. */

The label field will be ignored and standard value of 13 would be used. */
