Flowtracker Application

Flowtracker Overview

Flow Tracker is a mechanism that is used to track IP flows in a system and report the statistics, state changes to local or remote collector and apply policy. It tracks and collects the flow data and sends it to Flow collector. If flow collector is remote it sends the flow information via IPFIX. Flow Tracker keeps the state information about a flow and maintains statistics related to that flow. It can also perform metering for that flow. A flow is usually identified by 5 tuple (DIP, SIP, Protocol, D-PORT, S-PORT) and may include assigned objects such as Source Port, and VRF.

There are two modes of flowtracker in software.

  1. Hardware based flow tracking
  2. Software based flow tracking

In case of hardware based flow tracking, hardware performs the major flow tracking function. These functions include flow learing, flow ageing, flow exporting and maintaining flow state. Software configures hardware according to user specified configuration. Software also manages the hardware resources so it also acts as a resource manager for hardware assisted flow tracking. BCM56370 supports hardware supported flow tracking. Flowtracker version1 supports only one stage i.e Ingress Flowtracker, where as version2 supports flowtracking at multiple stages of pipelike i.e Ingress Flowtracker, MMU Flowtracker and Egress Flowtracker. BCM56275 supports version2 of hardware supported flow tracking.

In software based flowtracking, software manages flow learning, flow ageing, flow exporting and other functionalities. Flowtracker is an embedded application that runs on embedded ARM core 0 of BCM56960, BCM56970 & BCM56870 devices which learns incoming flows mapped to an user defined Flow Group. The flows are monitored by creating entries in the Exact Match table and attaching flex counters to it. The learnt flows are periodically packetized and sent to a collector in the IPFIX format (RFC 7011) as defined by the user. The app performs an aging process by which dormant flows are periodically removed from the Exact Match table.

The SDK APIs described below achieve configuring of flow groups, observation domain(s), reachability information of collector(s), IPFIX template(s) used for exporting packets to the collector and monitoring elephant flows.

Flow group tracking parameter configuration

Flow Group can track multiple packet attributes. These packet attributes are called tracking parameters. User sets these tracking parameters into the group. These section describes the methods that can be used to set/get tracking parameter in group. The tracking parameter can be part of flow key or flow data. A flag is provided in configuration to mark tracking parameter as key or data.

Flowtracker chip debug counter configuration

Flowtracker can be configured to provide a chip wide debug mode in case user wants to analyze some flows across the whole chip. User will have to set soc property flowtracker_chip_debug_enable to enable that feature. All the resource allocation management changes in software once this mode is enabled. This mode is not recommended for normal operation and should be only used for debugging. Once that mode is enabled, user can analyze counters on per tracking parameter. During bcm_flowtracker_check_create user passes tracking parameters on which check will be created. If this tracking parameter is currently in debugging mode, then all the checks created on this tracking meter will be added in same hardware memory. The counters are per hardware memory and then will be incremented whenever any check in that hardware memory passes. Thus counter can specify how many checks passed on that tracking parameter in whole asic.
