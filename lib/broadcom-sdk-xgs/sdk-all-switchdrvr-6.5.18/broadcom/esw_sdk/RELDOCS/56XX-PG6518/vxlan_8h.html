bcm_vxlan_port_add adds a Access or Network VXLAN port

Add VXLAN/NIV/PE port to VXLAN Network. VXLAN Network-domain ports are shared by all VXLAN VPNs. Customers should make use of a Reserved VXLAN VPN to add all Network-domain VXLAN ports within their system. When parameter vxlan_port->vxlan_port_id is NIV/PE port id and flag BCM_VXLAN_PORT_REPLACE is used, this API is used to add the NIV/PE port to VxLAN vpn forwarding domain. But please note that the NIV/PE port is not converted to VxLAN port, it's still NIV/PE port. NIV/PE port id should be used for virtual port operations. When invalid vpn id BCM_VXLAN_VPN_INVALID and vxlan port match criteria BCM_VXLAN_PORT_MATCH_SHARE are used, this API is used to create access shared virtual port, which does not belong to any VPN and can be shared by multiple VXLAN VPNs. After the shared logical port is created, bcm_port_match_add() can be used to associate multiple VPNs with that shared virtual port. Before deleting the shared logical port, we should use bcm_port_match_delete() to remove all the associated port match criteria and VPNs first of all. The port match added by bcm_port_match_add() should be deleted by bcm_port_match_delete().

Get counter statistic values for a vxlan vp/vpn index.

This API will retrieve set of counter statistic values for a vxlan vp/vpn index. The below table mentions the flex stat actions for the input vxlan parameters.

Flex Stat Actions for Input VXLAN Parameters
VxlanPort Vpn flex stat Action
1. ValidPort -1 Port stat counter
2. -1 Validvpn Vpn stat counter
3. ValidPort ValidVpn Port+Vpn stat counter
4. InvalidPort InvalidVpn return BCM_E_PARAM

Usage example: Assume user calls bcm_stat_group_create() API with mode type bcmStatGroupModeTrafficType. This creates a group of three counters with base_index for unicast packet, base_index+1 for multicast packet and base_index+2 for broadcast packet. If user wants to retrieve all 6 counters, 3 packet counters and 3 byte counters, then this API should be called twice, once for 3 packet counters and once for 3 byte counters. Here is the code sequence:

bcm_vxlan_stat_t stat;
uint32 cnt_index[3];
bcm_stat_value_t cnt_value[3];
int ucast_index=0;
int mcast_index=1;
int bcast_index=2;
cnt_index[0] = ucast_index;
cnt_index[1] = mcast_index;
cnt_index[2] = bcast_index;
// first time, get all three packet counters
stat = bcmVxlanInPackets;
bcm_vxlan_stat_counter_get(unit,port,vpn,stat,3,cnt_index,cnt_value);
// second time, get all three byte counters
stat = bcmVxlanInBytes;
bcm_vxlan_stat_counter_get(unit,port,vpn,stat,3,cnt_index,cnt_value);
// With the current cnt_index[] assignments, cnt_value[] are mapped as: cnt_value[0] --> ucast_counters(packets and bytes) cnt_value[1] --> mcast_counters(packets and bytes) cnt_value[2] --> bcast_counters(packets and bytes)
// User can reorder the cnt_value[] by reordering the cnt_index[]. For example, assign cnt_index[] as:
cnt_index[0] = bcast_index;
cnt_index[1] = mcast_index;
cnt_index[2] = ucast_index;
// Now cnt_value[] are mapped as: cnt_value[0] --> bcast_counters(packets and bytes) cnt_value[1] --> mcast_counters(packets and bytes) cnt_value[2] --> ucast_counters(packets and bytes)
