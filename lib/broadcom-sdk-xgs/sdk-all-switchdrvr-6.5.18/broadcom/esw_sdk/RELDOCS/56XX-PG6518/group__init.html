◆ bcm_attach()

int bcm_attach ( int  unit,
char *  type,
char *  subtype,
int  remunit 
)

Attach a device as a BCM unit.

Create a BCM unit instance. If the unit parameter is non-negative, use that value as the unit number for the attached device. Otherwise, search for an unused BCM unit number. If the type parameter is NULL, then search for an appropriate dispatch driver.

BCM API Dispatch Drivers
Type Name Description
loop Loopback Provide an alternate BCM unit number to access an underlying device. remunit is the underlying BCM unit number.
client Remote Procedure Call Access to a BCM unit attached to a remote CPU. remunit is the BCM unit number on the remote CPU. subtype is the CPU key (often a MAC address) to indicate with which remote CPU to communicate.
esw Enterprise Switch A locally attached StrataXGS device. remunit is the unit for the underlying driver layer (the SoC layer).
Parameters
[in,out]unitUnit number.
[in,out]typeType of BCM API dispatch driver
[in,out]subtypeArgument to dispatch driver
[in,out]remunitUnderlying remote unit
Return values
UNITthe attached non-negative unit number (if successful)
BCM_E_MEMORYmemory allocation failure
BCM_E_CONFIGdispatch driver not found
BCM_E_FULLno available unit numbers
BCM_E_EXISTSrequested unit number already attached
BCM_E_XXX

◆ bcm_attach_early_txrx()

int bcm_attach_early_txrx ( int  unit,
char *  type,
char *  subtype,
int  remunit 
)

Attach only tx and rx modules to a device as a BCM unit.

This API is primarily for warmboot. Since regular attach operation through bcm_attach during warmboot take a while to complete, there is a chance that protocols running in application layer time out. This can lead to disruption at system level. This API enables applications to start Tx and Rx as soon as possible, with some limitations. The limitations with initializing tx include, but are not limited to the following: a. Device doesnot check the link state in case of TX, and thus it might not free up the dma structures allocated to packet tx, if link is down. The limitations with initializing Rx include, but are not limited to the following: a. Trunk group gport might not be resolved, for packets received on trunk member ports After soc level initialization, this API can be called to initialize Tx and Rx modules. A regular bcm_attach must be followed for complete initialization of the device. Apart from Tx and Rx APIs, other APIs shouldnt be used/called before bcm_attach completes.

Parameters
[in,out]unitUnit number.
[in,out]typeType of BCM API dispatch driver
[in,out]subtypeArgument to dispatch driver
[in,out]remunitUnderlying remote unit
Return values
UNITthe attached non-negative unit number (if successful)
BCM_E_MEMORYmemory allocation failure
BCM_E_CONFIGdispatch driver not found
BCM_E_FULLno available unit numbers
BCM_E_EXISTSrequested unit number already attached
BCM_E_XXX

◆ bcm_attach_register()

int bcm_attach_register ( int  unit,
bcm_attach_cb_t  cb,
void *  user_data 
)

Register/unregister a callback function to be called before BCM device attachment and after BCM device detachment.

Register/unregister a callback function to be called in bcm_attach and bcm_detach APIs. When bcm_attach is called, the registered callback routine is called before BCM device attachment. When bcm_detach is called, the registered callback routine is called after BCM device detachment. This callback will be called under a lock, so will not be called re-entrantly. This allows the calling application to call SOC and other BCM APIs as appropriate. Registering/unregistering for a unit less than zero will operate in all BCM units, otherwise operation will be applied for a specific BCM unit.

Parameters
[in,out]unitUnit number.
[in,out]cbCallback function to register/unregister
[in,out]user_dataArbitrary application value passed to callback function

◆ bcm_detach()

int bcm_detach ( int  unit)

Detach a device as a BCM unit.

Destroy a BCM unit instance. The unit is removed from the list of BCM dispatchable units. Detaching a unit does not affect the underlying device if it was attached as a loopback or remote unit. This API is not thread-safe. In a multi-threaded environment where multiple threads may be calling various BCM APIs, it is recommended that such threads be stopped before invoking bcm_detach().

Parameters
[in,out]unitUnit number.
Return values
BCM_E_XXX

◆ bcm_detach_late_txrx()

int bcm_detach_late_txrx ( int  unit)

Detach all modules excluding tx and rx from BCM unit.

In cases when stopping Tx and Rx needs to be delayed, this API can be used to detach all other modules apart from tx and rx. Tx and Rx will continue to work, (with limitations similar to the ones mentioned for bcm_attach_early_txrx) even after this API is invoked. After this API, customers are expected to shut down their Tx/Rx components, and invoke bcm_detach, to shutdown the system completely.

[in,out]unitUnit number.
Return values

◆ bcm_detach_retry_get()

int bcm_detach_retry_get ( int  unit,
bcm_detach_retry_t *  retry 
)

Set/get the poll interval and number of retries before detach exits.

Set/get the poll interval time between retries and the number of retries before bcm_detach exits. The detach process checks whether there is any thread executing an API on the unit before it proceeds to unit detachment. If there is any API currently being executed, it tries again after waiting for the specified poll interval time. The default detach retry time is 5 minutes, with poll_usecs set to 100000 usecs, and num_retries set to 3000. These default values can be overridden during compile time with the following compiler flags, BCM_DETACH_POLL_INTERVAL_USECS_DEFAULT and BCM_DETACH_NUM_RETRIES_DEFAULT. Setting for a unit less than zero will operate in all BCM units, otherwise operation will be applied for a specific BCM unit. On a "_set" a NULL "retry" parameter will set back the default value.

Parameters
[in,out]unitUnit number.
[in,out]retry(for "_set", IN) Detach retry information to set
(for "_get", OUT) Returns detach retry information

◆ bcm_init()

int bcm_init ( int  unit)

Initialize the BCM software layer for a device.

Initializes each BCM API module that is appropriate for the device type. As a special case, if bcm_init is called on a unit that is not yet attached, then bcm_attach is called on that unit before calling the dispatch driver specific bcm_init routine. This allows code that simply calls bcm_init on local units to auto-attach those local units. bcm_init() is automatically called from bcm_attach(), so there is no need to call this explicitly after an attach. It is recommended to only use bcm_attach() as bcm_init() is deprecated.

Parameters
[in,out]unitUnit number.
Return values
BCM_E_XXX
