Statistics

General

The BCM statistics module allows for retrieval of common SNMP statistics for use with switch management and diagnostics software. Both 64-bit and 32-bit counters are supported. In addition, statistics are provided for the following groups: RFC 1213 (MIB-II), RFC 1493 (Bridge MIB), RFC 1757 (EtherStat), RFC 2665 (Ethernet-Like MIB), RFC 2233 (Interfaces group). For more information, see the relevant RFCs.

Drop Event Counter

In Trident4 chip, drops are a special type of event as they can cause packet processing steps to be skipped in the pipeline. There is a dedicated counter attached to each drop event. Drop event counter API allows user to get/clear counters associated with drop events defined in <bcm/types.h>.

Trace Event Counter

In Trident4 chip, a trace event indicate that some specific logic was hit during processing. A 48 bits bitmap is used to carry trace events in the pipeline. There are 48 dedicated trace events counters, one per trace event. All 48 counters may be updated per-packet at the end of the pipeline. Trace event counter API allows user to get/clear counters associated with trace events defined in <bcm/types.h>.

API Brief

ADJUSTABLE_FLEXIBLE_COUNTER_s

Adjustable Flexible Counter

bcmStatGroupModeTrafficType will help to get the counters based on UC/BC/MC traffic types. To use this one has to create a group with bcmStatGroupModeTrafficType which will consume 3 indexes at ingress and 2 indexes at egress. However if user want to count specific traffic type like only UC at ingress then other two indexes would be a waste of resources.

Given that Flex counters are very few in number (depending upon chip Few K in Ingress and Few K in Egress side) , user would surely like to optimize stats counter usage.

New Flexible stat group APIs provide this flexibility completely to user and user can create statistic group in its own way and attach to different accounting objects.
