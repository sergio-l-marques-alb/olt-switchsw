◆ bcm_collector_export_profile_create()

int bcm_collector_export_profile_create ( int  unit,
uint32  options,
int *  export_profile_id,
bcm_collector_export_profile_t *  export_profile_info 
)

Creates an export profile with given export information.

This API creates a collector export profile with the information present in the export_profile_info structure. Size of the export packet can be set by either using the max_packet_length or the num_records field. max_packet_length indicates the maximum possible size of the export packet. num_records indicates the number of records that must be packed in a packet before it can be exported. Only one can be set at a time, the selection is based on the flag BCM_COLLECTOR_EXPORT_PROFILE_FLAGS_USE_NUM_RECORDS. Currently only telemetry module supports the num_records field and there it indicates the number of port stats that must be packed in a packet.

Parameters
[in,out]unitUnit number.
[in,out]optionsOptions bitmap of BCM_COLLECTOR_EXPORT_PROFILE_XXX.
[in,out]export_profile_id(IN/OUT) The ID of collector export created. IN parameter if options parameter has BCM_COLLECTOR_EXPORT_PROFILE_WITH_ID or BCM_COLLECTOR_EXPORT_PROFILE_REPLACE option set.
[in,out]export_profile_infoPointer to the collector export information structure.

Register callback routine for Local collector record export.

API to register a user defined callback function (alongwith a cookie) to be called everytime there is an export record available from hardware to be evicted to user. The registered callback function is required to return appropriate return codes upon processing the data record.

Parameters
[in,out]unitUnit number.
[in,out]collector_idCollector Software ID.
[in,out]callback_optionsCollector export callback options.
[in,out]callback_fnCallback function to call when data record is present.
[in,out]userdataCookie passed from registering API's context to the callback function.
Return values
BCM_E_XXX
