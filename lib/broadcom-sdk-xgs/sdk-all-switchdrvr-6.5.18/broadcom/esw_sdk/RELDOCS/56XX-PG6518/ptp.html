Precision Time Protocol

For devices which support them, the PTP API provides the means for the control and configuration of PTP stack instances.

The following PTP features are supported and configurable per PTP stack instance:

PTP Data Types

  • bcm_ptp_time_type_t is an enumeration of PTP Time of Day types
  • bcm_ptp_time_source_t is an enumeration of PTP time source type.
  • bcm_ptp_clock_accuracy_t is an enumeration of PTP clock accuracy types
  • bcm_ptp_cb_type_t is an enumeration of PTP callback types
  • bcm_ptp_clock_type_t is an enumeration of PTP clock types
  • bcm_ptp_tod_source_t is an enumeration of PTP TOD source
  • bcm_ptp_tod_format_t is an enumeration of PTP TOD formats
  • bcm_ptp_source_type_t is an enumeration of PTP Source types
  • bcm_ptp_delay_mechanism_t is an enumeration of PTP Delay Mechanism types
  • bcm_ptp_message_type_t is an enumeration of 1588 messages type according to 2008 standard
  • bcm_ptp_tlv_type_t is an enumeration of 1588 TLV types per IEEE Std. 1588-2008.

PTP Subsystem API

These API functions are subsystem-wide, and are not particular to a specific PTP Clock or Clock Port.

API Brief
bcm_ptp_init(int unit) Initializes the PTP subsystem
bcm_ptp_detach(int unit) Shut down the PTP subsystem

PTP Stack API

A PTP Stack corresponds to a single processor that parses and generates IEEE-1588 messages.

If the processor is external to the switch unit, an associated structure contains the details for how the host CPU can exchange Ethernet traffic with the stack processor, and how the host can access the processor via PCIe.

A stack processor that is internal to the switch unit has predefined transport mechanisms.

PTP Clock API

A PTP clock is the instantiation of a IEEE-1588 clock. A clock can have a single clock port (an Ordinary Clock), or have multiple clock ports (a Boundary Clock). A clock may act as a master or a slave, depending on the clock priority and clock class, as per the IEEE-1588 standard.

PTP Unicast Master Table

Unicast masters can establish communication with a slave in one of two ways: Signaled, and Static.

The signaling, or dynamic, method is initiated by the slave sending a PTP Signaling message to the master, and is the method that is standards-compliant. The static method is done by API configuration of the master, and does not involve Signaling messages. For both static and signaled configuration, a master entry must be added on the slave side, to provide the appropriate L2 header for sending packets to the remote master.
