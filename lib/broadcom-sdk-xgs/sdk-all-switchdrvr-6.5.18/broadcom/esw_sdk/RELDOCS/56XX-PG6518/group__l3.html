Create an Egress forwarding object.

Create a unicast L3 Egress forwarding object on a physical port. On BCM56850, this API can be used to create additional types of L3 Egress objects:

  • A unicast L3 Egress object on a virtual port can be created by setting bcm_l3_egress_t.port field to the GPORT ID representing the virtual port. Other fields relevant to the creation of a unicast L3 Egress object on a physical port are also relevant here.
  • A multicast L3 Egress object on a physical port can be created by setting the BCM_L3_IPMC flag. Other relevant fields of the bcm_l3_egress_t structure are intf and multicast_flags. The trunk, module, and port fields should not be set.
  • A multicast L3 Egress object on a virtual port can be created by setting the BCM_L3_IPMC flag. Other relevant fields of the bcm_l3_egress_t structure are intf, multicast_flags, and port (should be set to the GPORT ID representing the virtual port).
  • A multicast L3 Egress object on a L2 Egress object can be created by setting the BCM_L3_IPMC flag. Other relevant fields of the bcm_l3_egress_t structure are intf, multicast_flags, and encap_id. An example of a L2 Egress object is a NIV egress object created by bcm_niv_egress_add . The encap_id field should be set to the NIV egress object ID, contained in bcm_niv_egress_t.egress_if. To add a multicast L3 Egress object to an IPMC group, first derive the object's multicast encapsulation ID using bcm_multicast_egress_object_encap_get, then add the multicast encapsulation ID to the IPMC group using bcm_multicast_egress_add .
