◆ bcm_link_change()

int bcm_link_change ( int  unit,
bcm_pbmp_t  pbmp 
)

Force a transient link down event on a port.

Force a transient link-down event to be recognized regardless of the current state of the physical link. If the link is up at the time of the call, both a link down and a subsequent link up event will be recognized. If traffic is flowing on the link at the time of the call, the affect on the flowing traffic is undefined. If the link is down at the time of the call, no action will be taken.

Parameters
[in,out]unitUnit number.
[in,out]pbmp

Wait for a set of links to recognize link up.

This function is primarily useful for diagnostics. It suspends the calling thread until all the links specified in pbm are active (link up). If not all ports link up in the allotted time, pbm is updated to reflect those that have link link and BCM_E_TIMEOUT is returned.

[in,out]usMaximum number of microseconds to wait

◆ bcm_linkscan_enable_get()

int bcm_linkscan_enable_get ( int  unit,
int *  us 
)

Enable and disable link scanning or set the polling interval.

Calling bcm_linkscan_enable_set with a non-zero scanning interval sets the scan interval starts the linkscan task if required. If the specified interval is zero, the linkscan task is stopped. For this reason, it is required to have link scan enabled and running with a non-zero interval even if all ports are operating in hardware linkscan mode. bcm_linkscan_enable_get returns the current scan interval. A returned scan interval of 0 indicates linkscan is disabled.

Parameters
[in,out]unitUnit number.
[in,out]us(for "_set") Minimum time between software link scan cycles in micro-seconds. 0 indicates linkscan is disabled.
(for "_get", OUT) Minimum time between software link scan cycles in micro-seconds. 0 indicates linkscan is disabled.

◆ bcm_linkscan_init()

int bcm_linkscan_init ( int  unit)

Initialize the link scanning subsystem.

Initialize the linkscan subsystem. Link scanning is not performed until enabled using bcm_linkscan_enable_set . The linkscan thread may be launched during this call, or when link scanning is enabled using bcm_linkscan_enable_set . Generally this routine need be called once during system initialization. However, there may be circumstances under which the linkscan subsystem must be re-initialized. If this routine is called more than once, all registered callbacks (see bcm_linkscan_register) are unregistered.

Parameters
[in,out]unitUnit number.

◆ bcm_linkscan_mode_get()

int bcm_linkscan_mode_get ( int  unit,
bcm_port_t  port,
int *  mode 
)

Get or set the link scanning mode for a port.

Get or set the current link scanning mode for a port. The possible modes for a port are described in table BCM_LINKSCAN_MODE_e . Note: Scan mode BCM_LINKSCAN_MODE_OVERRIDE is only supported on SDKLT-based devices. When a port is set to the scan mode BCM_LINKSCAN_MODE_OVERRIDE, linkscan will not poll the internal PHY link status of this port. Application has to provide the link state of this port through bcm_linkscan_override_link_state_set(). Linkscan will keep tracing the override link state of this port, and process link update when the override link state got changed. The typical usage of OVERRIDE linkscan mode is for the case that ports have external PHYs. In this case, application should set the port to OVERRIDE mode, monitor the external PHY link status by itself, and notify linkscan module with the external PHY link change when it occurs. When a port is set to BCM_LINKSCAN_MODE_NONE, linkscan does not process any link change on a port. This mode is only recommended if the application do not want to use the port (i.e. port is disabled). bcm_linkscan_mode_set operates on one port while bcm_linkscan_mode_set_pbm operates on all ports specified in the port bit map. When operating on a port bit map, a failure on any one port aborts the processing of the request and the link scan mode of all the ports specified is undefined. Ports not in the port bit map are unaffected. The use of bcm_linkscan_mode_set_pbm is not recommended.

Link Scanning Modes
Mode Meaning
BCM_LINKSCAN_MODE_HW Hardware link scanning
BCM_LINKSCAN_MODE_SW Software link scanning
BCM_LINKSCAN_MODE_OVERRIDE Override link scanning
BCM_LINKSCAN_MODE_NONE No link scanning.
Parameters
[in,out]unitUnit number.
[in,out]portDevice port number to set or get the mode of
[in,out]mode(for "_set" and "_set_pbm") Link scan mode, see table BCM_LINKSCAN_MODE_e .
(for "_get", OUT) Link scan mode, see table BCM_LINKSCAN_MODE_e .

Register and unregister link status callout.

Register a handler to be called when a link status is to be determined by a caller provided function. An example of use of this API is to determine the true link status for an optical signal where PHY registers might not give precise indication of link status. ◆ bcm_linkscan_register()

int bcm_linkscan_register ( int  unit,
bcm_linkscan_handler_t  f 
)

Register and unregister link notification callouts.

Register a handler invoked on a link up or link down transition. If multiple handlers are registered, they are invoked on a link transition in the same order as they were registered. The behavior of registering the same callout more than once is undefined. If an application chooses to perform custom port configurations which bypass API functions, such custom operations may need to be repeated whenever a port's link comes up. Consult a Field Application Engineer for guidance.

Parameters
[in,out]unitUnit number.
[in,out]fBCM callback function pointer (see bcm_linkscan_handler_t)
